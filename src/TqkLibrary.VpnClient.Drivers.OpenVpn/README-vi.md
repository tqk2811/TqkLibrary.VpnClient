# TqkLibrary.VpnClient.Drivers.OpenVpn

Driver **OpenVPN** (tương thích OpenVPN community server) — ráp các khối protocol thuần ở [`OpenVpn`](../TqkLibrary.VpnClient.OpenVpn) (V2.a–V2.g) thành một tunnel chạy thật sau facade. Chọn **transport UDP** (1 datagram = 1 gói) hoặc **TCP** (16-bit length framing) theo `proto`, **demux opcode** control↔data (cả **P_DATA_V1 lẫn P_DATA_V2**) trên cùng một transport, chạy control-channel handshake (reset → TLS → key-method-2 → PUSH) rồi `BuildDataChannel` chọn **AEAD** (AES/ChaCha-GCM) hoặc **non-AEAD CBC** (AES-CBC + HMAC `--auth`, cho server NCP-less như SoftEther) cắm vào device-link. **`dev tun`** (gói IP trần → `IPacketChannel`) bind thẳng L3; **`dev tap`** (khung Ethernet → `IEthernetChannel`) bắc cầu xuống `IPacketChannel` qua **fabric L2 userspace** với **3 chế độ**: (a) **1-host theo ifconfig** (`OpenVpnTapChannel` → `ArpResolver` + `VirtualHost`, IP từ `ifconfig` server-bridge push); (b) **pure-DHCP** (server không push ifconfig → [`DhcpV4Configurator`](../TqkLibrary.VpnClient.Ethernet/DhcpV4Configurator.cs) L2.5 lease IP trên kênh tap); (c) **multi-host** (opt-in `multiHost`: tap gắn vào [`EthernetAdapter.ConnectUplink`](../TqkLibrary.VpnClient.Ethernet/EthernetAdapter.cs) như uplink port + [`MultiHostSession`](../TqkLibrary.VpnClient.Ethernet/MultiHostSession.cs), thêm station qua `OpenSessionAsync`, mỗi station DHCP lease riêng). IPv4/ARP mặc định; **IPv6 trong tunnel opt-in `enableIpv6`** (chỉ tap): chạy thêm **NDISC v6** (L2.4) + **SLAAC/DHCPv6** (L2.6) gom qua `DualStack*` (L2.9) trên cùng segment L2, best-effort (bridge IPv4-only vẫn kết nối) — **mirror SoftEther V.4-ipv6**.

## Vị trí kiến trúc

`PROTOCOL`-layer driver, hiện thực [`IVpnProtocolDriver`](../TqkLibrary.VpnClient.Abstractions/Drivers/Interfaces/IVpnProtocolDriver.cs). Lắp ráp các khối protocol thuần từ [`OpenVpn`](../TqkLibrary.VpnClient.OpenVpn) thành 1 tunnel sống:

- **Transport**: [`OpenVpnUdpTransport`](../TqkLibrary.VpnClient.OpenVpn/Transport/OpenVpnUdpTransport.cs) (UDP) / [`OpenVpnTcpTransport`](../TqkLibrary.VpnClient.OpenVpn/Transport/OpenVpnTcpTransport.cs) (TCP, bọc **shared** [`TcpByteStream`](../TqkLibrary.VpnClient.Transport.Tcp/TcpByteStream.cs#L22) trần F.1) — driver dựng socket thật qua [`OpenVpnSocketTransportFactory`](Transport/OpenVpnSocketTransportFactory.cs) (seam `IOpenVpnTransportFactory`, test inject loopback).
- **Control plane**: [`OpenVpnControlChannel`](../TqkLibrary.VpnClient.OpenVpn/OpenVpnControlChannel.cs#L22) (reset HARD_RESET → `SslStream` trong reliability layer → key-method-2 → PUSH_REQUEST/PUSH_REPLY); wrap tls-auth/tls-crypt opt-in (`IOpenVpnControlWrap`).
- **Data plane**: [`OpenVpnDataPlane`](../TqkLibrary.VpnClient.OpenVpn/DataChannel/OpenVpnDataPlane.cs#L10) (P_DATA_V2 + AES-GCM, make-before-break) qua device-link — **tun** [`OpenVpnTunChannel`](../TqkLibrary.VpnClient.OpenVpn/Transport/OpenVpnTunChannel.cs) → `IPacketChannel` trực tiếp; **tap** [`OpenVpnTapChannel`](../TqkLibrary.VpnClient.OpenVpn/Transport/OpenVpnTapChannel.cs) → (1-host/pure-DHCP) [`VirtualHost`](../TqkLibrary.VpnClient.Ethernet/VirtualHost.cs) + [`ArpResolver`](../TqkLibrary.VpnClient.Ethernet/ArpResolver.cs) (+ [`DhcpV4Configurator`](../TqkLibrary.VpnClient.Ethernet/DhcpV4Configurator.cs) khi không ifconfig) / (multi-host) [`EthernetAdapter.ConnectUplink`](../TqkLibrary.VpnClient.Ethernet/EthernetAdapter.cs) + [`MultiHostSession`](../TqkLibrary.VpnClient.Ethernet/MultiHostSession.cs) → `IPacketChannel`.

## Luồng kết nối (as-built)

1. **Resolve** host qua [`IHostResolver`](../TqkLibrary.VpnClient.Abstractions/Net/IHostResolver.cs) theo `AddressFamilyPreference`; IP-literal verbatim không-DNS.
2. **Transport**: `IOpenVpnTransportFactory.ConnectAsync(IPEndPoint, ct)` dựng socket (UDP/TCP) → `OpenVpnUdpTransport`/`OpenVpnTcpTransport`. Sub `OnTransportDatagram` (P_DATA_V2) + ctor `OpenVpnControlChannel` (control) gắn vào cùng `DatagramReceived` = **demux opcode**; chạy receive-pump nền (socket transport) — loopback tự pump.
3. **Reset + TLS** (`OpenVpnControlChannel.ConnectAsync`): HARD_RESET_CLIENT_V2 ⇄ HARD_RESET_SERVER_V2 (reliability id 0) → `SslStream` AuthenticateAsClient **bên trong** reliability layer (validate server cert qua callback; client cert tùy chọn).
4. **key-method-2** (`NegotiateKeyMaterialAsync`): gửi key_source2 + **peer-info** trên TLS (NCP `IV_CIPHERS` + **push-peer-info nâng cao** qua [`OpenVpnPeerInfoOptions`](../TqkLibrary.VpnClient.OpenVpn/DataChannel/OpenVpnPeerInfo.cs#L85) ở ctor — `IV_MTU` mặc định lấy tun MTU khi caller chưa đặt, cộng IV_PLAT/IV_PLAT_VER/IV_SSL_VER/IV_GUI_VER + user-var `UV_*`), đọc reply server → `OpenVpnKeyMaterial` (key2 256B, cipher-independent).
5. **PUSH** (`RequestConfigAsync`): PUSH_REQUEST → PUSH_REPLY → `OpenVpnPushReply` (ifconfig/route/DNS/peer-id/ping/cipher/topology). Không có ifconfig: **tun** ⇒ `VpnServerRejectedException`; **tap** ⇒ rơi xuống pure-DHCP (chạy `DhcpV4Configurator` lease IP trên kênh tap, L2.5).
6. **NCP + data plane** ([`BuildDataChannel`](OpenVpnConnection.cs#L317)): cipher = (1) server chốt trong PUSH `cipher`; (2) server **không** push `cipher` ⇒ khai thác `IV_CIPHERS` server đẩy ngược trong OCC options của key-method-2 reply (`control.ServerKeyMethodOptions`); (3) fallback cipher cấu hình (`fallbackCipher` = `cipher`/`data-ciphers[0]`); (4) mặc định AES-256-GCM. Cipher **AEAD** (GCM/ChaCha) ⇒ `keyMaterial.DeriveDataKeys(cipher)` → `OpenVpnDataChannel`; cipher **AES-CBC** (server NCP-less, vd SoftEther) ⇒ `keyMaterial.DeriveCbcDataKeys(cbc, OpenVpnDataAuth.KeySizeBytes(dataAuth))` → `OpenVpnCbcDataChannel` với HMAC `--auth` (`dataAuth = profile.Auth`, mặc định SHA1); cipher không hỗ trợ ⇒ `VpnConnectionException`. **P_DATA_V2 khi `push.PeerId.HasValue`, ngược lại P_DATA_V1**. Cả 2 channel sau `IOpenVpnDataChannel` → `OpenVpnDataPlane`; compression từ PUSH (chỉ no-compression).
7. **Bind device-link**: **tun** → `OpenVpnTunChannel(dataPlane, compression, sink=transport.SendAsync, mtu)` → `_facade.SetInner(tun)`. **tap** → `OpenVpnTapChannel(..., mac)` (MAC locally-administered tự sinh, giữ qua reconnect), `_dataActive=true` để DHCP/ARP chạy được, rồi: **(1-host)** `BridgeTapSingleHostAsync` — ifconfig sẵn ⇒ dùng luôn, không-ifconfig ⇒ `DhcpV4Configurator` lease (pure-DHCP), rồi `ArpResolver`(địa chỉ) + `VirtualHost` → `_facade.SetInner(host)`; **(multi-host, `multiHost: true`)** `BridgeTapMultiHostAsync` — `EthernetAdapter.ConnectUplink(tap)` + `MultiHostSession`, primary station (ifconfig tĩnh hoặc DHCP) → `_facade.SetInner(primary.PacketChannel)`, `OpenSessionAsync` thêm station DHCP. IP stack vẫn bind `IPacketChannel`, MTU = link−14; ifconfig/station phải IPv4 (ARP), v6 ⇒ `VpnConnectionException` trỏ **L2.4** NDISC. `TunnelConfig` từ PUSH/DHCP; bật keepalive.

## Vòng đời

- **Demux opcode**: mỗi datagram vào cả `OpenVpnControlChannel.OnDatagram` (chỉ control, `TryDecodeControl`) lẫn `OnTransportDatagram` (cả `P_DATA_V1` **và** `P_DATA_V2`, lọc theo `ReadOpcode`) — mỗi handler tự bỏ gói không thuộc về mình. Gói data inbound → đếm keepalive-received → `OpenVpnTunChannel.Deliver` (giải mã + de-compress + **drop ping** + raise `InboundIpPacket`). Gói data outbound đi thẳng `sink`→`transport.SendAsync` (không qua `DatagramReceived`).
- **Sink an toàn qua reconnect-window (sửa crash)**: data-plane sink đọc `_transport` **hiện tại** + bỏ qua khi `!_dataActive` (đang reconnect) + nuốt `ObjectDisposedException`/`SocketException` ([sink @ OpenVpnConnection.cs:273](OpenVpnConnection.cs#L273)). Trước đây một RTO-retransmit của TCP-stack bắn vào socket vừa dispose ⇒ unhandled trong timer callback ⇒ sập process; nay drop êm (TCP retransmit lại sau). [`OpenVpnSocketTransportFactory.UdpDatagramSocket`](Transport/OpenVpnSocketTransportFactory.cs#L46) giữ ref `Socket` (tránh `UdpClient.Client` null khi dispose đua) + tắt **`SIO_UDP_CONNRESET`** trên Windows (ICMP port-unreachable không còn làm receive/send sau đó ném `ConnectionReset`).
- **Keepalive** (ping/ping-restart từ PUSH): timer 1s — `OpenVpnKeepalive.ShouldSendPing` ⇒ gửi `OpenVpnPing.Magic` qua data plane (đếm sent), `IsPeerDead` ⇒ link lost. Gói data ra của user cũng reset ping-send timer. Cả 2 = 0 ⇒ không bật timer.
- **Rekey**: `OpenVpnDataPlane.RekeyNeeded` (packet-id tới ~2³²) ⇒ **re-establish** (reconnect) — fresh keys, packet-id về 0, không tái dùng nonce GCM. (Soft-reset make-before-break — TLS handshake thứ 2 trên key_id mới rồi `OpenVpnDataPlane.Swap` — là việc sau, cần control-channel hỗ trợ SOFT_RESET.)
- **Teardown** (`OpenVpnConnection.DisconnectAsync` override): gửi **explicit-exit-notify** (`EXIT_NOTIFY` qua [`OpenVpnControlChannel.SendExitNotifyAsync`](../TqkLibrary.VpnClient.OpenVpn/OpenVpnControlChannel.cs)) best-effort + time-box 2s để server drop session ngay (không chờ keepalive timeout), rồi gọi `DisconnectCoreAsync` của base: hủy reconnect đang chờ, hủy receive loop, dispose control channel + socket (`CleanupAttemptResourcesAsync` của driver). Exit-notify không bao giờ chặn/làm fail teardown.
- **Reconnect** (supervisor F.6, gom về [`ReconnectingVpnConnection<…>`](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs#L24)): base sở hữu `Facade`/lifetime CTS/cờ `IsRunning`/máy link-loss→supervisor→`ReconnectLoopAsync` (backoff+jitter)/`SetState`/clock; driver chỉ override `EstablishAsync`/`CleanupAttemptResourcesAsync`/`StopAttemptLoop` + 4 ánh xạ state + `OnReconnected` (raise `Reconnected` → `OpenVpnVpnSession.ApplyReconnect`). Rekey re-establish và ping-restart đều gọi `OnLinkLost` của base.
- **Logging/diagnostics (Q.2)**: ctor `OpenVpnConnection`/`OpenVpnDriver` nhận `ILoggerFactory?` (mặc định [`NullLogger`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs) ⇒ no-op, **ADDITIVE không đổi hành vi**). Log qua [`VpnLogExtensions`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs): reset/TLS + key-method-2 + PUSH_REPLY→Handshake (PUSH thiếu ifconfig→HandshakeFailed); bind data plane→HandshakeCompleted; ping→Keepalive; rekey re-establish→Rekey; `SetState`→StateChanged; `OnLinkLost`→LinkLost; reconnect attempt/success→ReconnectAttempt/Reconnected.

## Bảng chuẩn / RFC

| Khối | Chuẩn | Ghi chú |
|------|-------|---------|
| Control reliability + TLS-in-control | OpenVPN wire protocol | reset/P_CONTROL/P_ACK + key-method-2 (xem [OpenVpn README](../TqkLibrary.VpnClient.OpenVpn/README-vi.md)) |
| Data channel AEAD | P_DATA_V1/V2 + AES-256/128-GCM + CHACHA20-POLY1305 | NCP `data-ciphers` đàm phán qua `IV_CIPHERS`/PUSH `cipher` |
| Data channel non-AEAD | P_DATA_V1/V2 + AES-128/192/256-CBC + HMAC (`--auth`) | server NCP-less (vd OpenVPN function của SoftEther); HMAC full-digest SHA1/256/384/512/MD5 |
| Transport UDP/TCP | OpenVPN `proto udp`/`proto tcp` | UDP 1 datagram = 1 gói; TCP 16-bit length framing (seam F.2) |
| TLS PRF key derivation | TLS 1.0 PRF | `tls-ekm` (RFC 5705) chờ **F.5** |
