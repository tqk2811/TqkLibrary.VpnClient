# TqkLibrary.VpnClient.Ethernet

> Tầng L2 Ethernet userspace — codec khung Ethernet II ([EthernetFrame.cs](EthernetFrame.cs)) + địa chỉ MAC 48-bit ([MacAddress.cs](MacAddress.cs)) + **switch học MAC** in-memory ([EthernetSwitch.cs](EthernetSwitch.cs)) + **`VirtualHost` cầu nối L2↔L3** ([VirtualHost.cs](VirtualHost.cs)) + **ARP IPv4** (`INeighborResolver`: [ArpResolver.cs](ArpResolver.cs) + codec [ArpPacket.cs](ArpPacket.cs)) + **NDISC IPv6** (`INeighborResolver`: [NdiscResolver.cs](NdiscResolver.cs) + codec [Icmpv6Ndisc.cs](Icmpv6Ndisc.cs)) + **DHCPv4 client** (`IAddressConfigurator`: [DhcpV4Configurator.cs](DhcpV4Configurator.cs) + codec [DhcpV4Packet.cs](DhcpV4Packet.cs)/[DhcpV4Options.cs](DhcpV4Options.cs)) + **SLAAC + DHCPv6 client IPv6** (`IAddressConfigurator`: [Ipv6AddressConfigurator.cs](Ipv6AddressConfigurator.cs) + codec [Dhcpv6Packet.cs](Dhcpv6Packet.cs)/[Dhcpv6Options.cs](Dhcpv6Options.cs) + helper [Helpers/SlaacAddress.cs](Helpers/SlaacAddress.cs)) + **`EthernetAdapter` ráp multi-host** ([EthernetAdapter.cs](EthernetAdapter.cs) + handle [EthernetAdapter.Host.cs](EthernetAdapter.Host.cs) + backpressure [BackpressuredPacketChannel.cs](BackpressuredPacketChannel.cs) + spec [Models/EthernetHostSpec.cs](Models/EthernetHostSpec.cs)) + **multi-host session model** ([MultiHostSession.cs](MultiHostSession.cs) + station [EthernetHostSession.cs](EthernetHostSession.cs)) + **composite dual-stack** ([DualStackNeighborResolver.cs](DualStackNeighborResolver.cs) — route ARP/NDISC theo `AddressFamily`; [DualStackAddressConfigurator.cs](DualStackAddressConfigurator.cs) — merge DHCPv4 + SLAAC/DHCPv6 vào một `TunnelConfig`). **Hoàn chỉnh phase L2.0–L2.9** — `EthernetAdapter` ráp switch + N×virtual host + ARP/NDISC/DHCP + backpressure thành LAN userspace; `MultiHostSession` phơi N×`VirtualHost` thành N×`IVpnSession` (= N "máy LAN") cho driver L2 (`VpnLinkLayer.L2Ethernet`/`MultiHostModel.L2BroadcastDomain`); composite dual-stack cho phép **một** host nói cả IPv4 lẫn IPv6 trên cùng `VirtualHost` (L2.9 chốt tầng L2: test multi-host dual-stack offline).

## Mục đích

Project này là **nền tầng L2** cho mô phỏng LAN ảo userspace (xem design [03-multihost-l2-vs-l3.md](../../.docs/03-multihost-l2-vs-l3.md) / [09-userspace-ipstack.md](../../.docs/09-userspace-ipstack.md)). Đã có **L2.0** (codec không-phụ-thuộc) + **L2.1** (switch học MAC) + **L2.2** (`VirtualHost` cầu nối L2↔L3) + **L2.3** (ARP IPv4 — `INeighborResolver` thật) + **L2.4** (NDISC IPv6 — `INeighborResolver` thật cho v6 + DAD + parse RA) + **L2.5** (DHCPv4 client — `IAddressConfigurator` thật, DISCOVER/OFFER/REQUEST/ACK → `TunnelConfig`) + **L2.6** (SLAAC + DHCPv6 client — `IAddressConfigurator` thật cho v6: SLAAC từ RA + DHCPv6 stateful SOLICIT/ADVERTISE/REQUEST/REPLY → `TunnelConfig` v6) + **L2.7** (`EthernetAdapter` ráp — compose switch + N×`VirtualHost` + resolver/configurator + backpressure → mỗi host một `IPacketChannel` độc lập); các phase L2.8→L2.9 cắm lên:

- **Codec khung Ethernet II** (`dst MAC | src MAC | EtherType | payload`) — đối xứng cách project anh em `TqkLibrary.VpnClient.IpStack` cung cấp codec `Ipv4`/`Ipv6`.
- **Kiểu địa chỉ MAC** (`MacAddress`) — value type 48-bit, đóng vai trò khoá FDB của switch (L2.1) và khoá neighbor cache (L2.3/L2.4).
- **`EthernetSwitch`** (L2.1) — switch phần mềm in-memory học `source MAC → port` (FDB) và forward theo dest MAC (unicast đã học đi 1 port; broadcast/multicast/unknown-unicast flood) — là "fabric" để cắm N `VirtualHost`. Hai loại port dưới base `SwitchPort`: **host-port** (`ConnectHost`) switch sở hữu, và **uplink-port** (`ConnectUplink(IEthernetChannel)`) bọc kênh VPN ngoài để bắc cả broadcast domain lên tunnel (switch flood/forward ra → `WriteFrameAsync`; `InboundFrame` của uplink → ingress vào switch) — đường data-plane đa-host.
- **`VirtualHost`** (L2.2) — mỗi "máy ảo" trên LAN = {MAC, port switch, resolver}, **expose một `IPacketChannel`** cho `TcpIpStack` đã có (dual-stack v4/v6): egress bọc gói IP vào khung Ethernet (resolve dest-MAC qua `INeighborResolver`) → switch; ingress tháo khung → gói IP cho stack. Đây là nơi **quy tắc vàng** thành hiện thực — stack chỉ thấy `IPacketChannel`.
- **`ArpResolver`** (L2.3) — hiện thực **`INeighborResolver` cho IPv4** (RFC 826): resolve next-hop IP→MAC khi `VirtualHost` egress (cache hit trả ngay; miss → broadcast ARP request rồi chờ reply, retry/timeout), học mọi gói ARP vào cache, **trả lời** ARP request hỏi IP của ta. Lắng nghe khung ARP inbound qua **seam `VirtualHost.InboundNonIpFrame`** (đẩy vào bằng `HandleInboundFrame`, composer nối — không tạo phụ thuộc vòng). Codec gói ARP tách ở `ArpPacket`.
- **`NdiscResolver`** (L2.4) — hiện thực **`INeighborResolver` cho IPv6** (RFC 4861, mirror `ArpResolver`): resolve next-hop IPv6→MAC khi `VirtualHost` egress — miss → gửi **Neighbor Solicitation** tới **solicited-node multicast** (`ff02::1:ffXX:XXXX`, MAC `33:33:ff:..`) rồi chờ **Neighbor Advertisement**; học IPv6→MAC từ NA (Target-LLA) + NS (Source-LLA), **trả lời** NS hỏi địa chỉ của ta bằng NA solicited. Thêm **DAD** (Duplicate Address Detection — gửi NS từ `::`, phát hiện trùng qua NA defend, RFC 4862 §5.4) + **parse RA** lấy gateway + prefix (`RouterAdvertisementInfo`, phục vụ SLAAC P1.1/L2.6). Vì NDISC ride **trong IPv6** (không EtherType riêng), composer nối **cả** `VirtualHost.InboundIpPacket` lẫn `InboundNonIpFrame`. Codec ICMPv6 NS/NA/RS/RA + option tách ở `Icmpv6Ndisc` (tự tính checksum pseudo-header RFC 8200 §8.1 — **không** ref `IpStack`).
- **`DhcpV4Configurator`** (L2.5) — hiện thực **`IAddressConfigurator` cho IPv4** (RFC 2131/2132): chạy handshake bốn bước **DISCOVER → OFFER → REQUEST → ACK** để xin lease IP từ DHCP server (SecureNAT của SoftEther / `server-bridge` của OpenVPN-tap). Gửi DISCOVER/REQUEST dạng **broadcast UDP/IPv4 68→67** — tự dựng header IPv4 + UDP (checksum pseudo-header RFC 768/791, **không** ref `IpStack`, mirror cách `Icmpv6Ndisc` build IPv6) rồi bọc vào khung Ethernet broadcast (`ff:ff:ff:ff:ff:ff`) → port switch. Lắng nghe reply qua **seam `VirtualHost.InboundIpPacket`** (DHCP ride trong IPv4 nên stack sẽ nuốt nếu không nối) + `InboundNonIpFrame` cho đầy đủ; **xid** (transaction id ngẫu nhiên) tương quan request↔reply + **retransmit** theo `DhcpV4ConfiguratorOptions` (timeout/MaxAttempts). Trả `TunnelConfig` (IP=yiaddr, prefix từ option 1 subnet-mask, default-route từ option 3 router, DNS từ option 6, MTU theo options); **NAK** (option 53=6) ⇒ ném; hết attempts ⇒ ném. Codec gói BOOTP/DHCP + UDP/IPv4 framing tách ở `DhcpV4Packet`; codec option (magic-cookie + TLV 53/50/54/51/1/3/6) tách ở `DhcpV4Options`.
- **`Ipv6AddressConfigurator`** (L2.6) — hiện thực **`IAddressConfigurator` cho IPv6** (SLAAC RFC 4862 + DHCPv6 stateful RFC 8415), bộ đôi IPv6 của `DhcpV4Configurator`. Chọn cơ chế theo **Router Advertisement** y như host thật: (a) **SLAAC** — từ RA có prefix **autonomous (A)** /64 (do `NdiscResolver` L2.4 parse ra `RouterAdvertisementInfo`), dựng địa chỉ global bằng cách ghép prefix + **interface identifier** (Modified EUI-64 RFC 4291 hoặc opaque ổn định RFC 7217, chọn qua `SlaacInterfaceIdentifierMode`); router quảng bá thành **gateway IPv6**; (b) **DHCPv6 stateful** — khi RA bật cờ **Managed (M)** (hoặc không có prefix SLAAC, hoặc `ForceDhcp`), chạy **SOLICIT → ADVERTISE → REQUEST → REPLY** qua UDP 546→547 tới multicast `ff02::1:2` để xin địa chỉ; cờ **Other (O)** kéo thêm DNS từ cùng exchange. RA tự nó do `NdiscResolver` parse — configurator đọc `LastRouterAdvertisement`, nếu chưa có thì gửi **Router Solicitation** tới `ff02::2` rồi chờ event `RouterAdvertisementReceived`. Reply DHCPv6 ride trong IPv6 nên composer nối `HandleInboundFrame` vào `VirtualHost.InboundIpPacket` (+ `InboundNonIpFrame`), giống DHCPv4. Trả `TunnelConfig` v6 (`AssignedAddressV6`/`PrefixLengthV6`, default-route `::/0 <router>`, DNS). Codec gói DHCPv6 + UDP/IPv6 framing tách ở `Dhcpv6Packet`; codec option (Client/Server-ID DUID-LL + IA_NA/IA-Address + Option-Request + Status + DNS) tách ở `Dhcpv6Options`; helper tạo địa chỉ SLAAC ở `Helpers/SlaacAddress`.
- **`EthernetAdapter`** (L2.7) — **lớp ráp**: compose `EthernetSwitch` + N×`VirtualHost` thành LAN userspace multi-host (design 09: `EthernetAdapter` = IPacketChannel-provider). `AddHost(mac, build)` connect port switch → callback `build(port)` dựng `EthernetHostSpec` {`INeighborResolver` (ARP/NDISC), optional `IAddressConfigurator` (DHCP/SLAAC), seam-hook `NonIpFrameHandler`/`IpPacketHandler`} chia sẻ cùng port → tạo `VirtualHost` + **nối đúng seam thủ công mà driver đang làm** (`host.InboundNonIpFrame += arp.HandleInboundFrame`; `host.InboundIpPacket += dhcp/ndisc.HandleInboundFrame`) → bọc `BackpressuredPacketChannel` → trả `EthernetHostHandle` (mỗi host một `IPacketChannel` độc lập `handle.Channel` + `ConfigureAsync` chạy DHCP/SLAAC). Là lớp để driver L2 (SoftEther/OpenVPN-tap) phơi broadcast domain đa-host thay vì bắc-cầu-1-host thủ công. **Backpressure** (design 09 §"Hiệu năng"): `BackpressuredPacketChannel` tách ingress của host khỏi consumer bằng **bounded `Channel<byte[]>` + một read-loop pump** (1 read-loop/`VirtualHost`) — switch deliver đồng bộ chỉ `TryWrite` vào queue (không block fabric), pump re-raise tới stack ngoài deliver-path, consumer chậm ⇒ `BoundedChannelFullMode` (mặc định `DropOldest`) bound queue.
- **Quy tắc vàng** (design [00 §5](../../.docs/00-architecture-overview.md)): stack TCP/IP **chỉ** bind `IPacketChannel`, không bao giờ thấy Ethernet; mọi MAC/ARP/ND/DHCP nằm ở tầng này. Vì vậy 2 *slot* `INeighborResolver` (ARP/NDISC) + `IAddressConfigurator` (DHCP/SLAAC) được khai báo trong `TqkLibrary.VpnClient.Abstractions` (dùng raw bytes/`IPAddress`, **không** ref `MacAddress`); `INeighborResolver` nay **đã có hiện thực IPv4 = `ArpResolver` (L2.3)** + **IPv6 = `NdiscResolver` (L2.4)**; `IAddressConfigurator` nay **đã có hiện thực IPv4 = `DhcpV4Configurator` (L2.5)** + **IPv6 = `Ipv6AddressConfigurator` (L2.6, SLAAC+DHCPv6)**.

## Vị trí trong kiến trúc

- **Tầng:** L2 (Ethernet) — nằm **dưới** `EthernetAdapter` tương lai; bắc cầu xuống `IPacketChannel` cho stack IP đã có.
- **Target frameworks:** `netstandard2.0; net8.0` (xem [src/Directory.Build.props](../Directory.Build.props)); `record`/`init` khả dụng cả 2 TFM nhờ polyfill `TqkLibrary.CompilerServices` (`IsExternalInit`) — `MacAddress` vẫn là `readonly struct` thường (as-built).
- **Phụ thuộc:**
  - ProjectReference: **chỉ** `TqkLibrary.VpnClient.Abstractions` (slot L2 `INeighborResolver`/`IAddressConfigurator` + `IEthernetChannel`/`IPacketChannel`/`LinkMedium` mà `VirtualHost` hiện thực/tiêu thụ). **Cố ý KHÔNG** ref `TqkLibrary.VpnClient.IpStack` dù `VirtualHost` cần đọc version-nibble + IP đích **và `NdiscResolver` cần build/parse IPv6+ICMPv6** — đó là **phụ thuộc ngang** mà layering cấm ([10 §2](../../.docs/10-codebase-architecture-and-flow.md)), nên đọc/ghi thẳng từ offset header cố định (RFC 791/8200) + tự tính checksum pseudo-header trong [`Icmpv6Ndisc`](Icmpv6Ndisc.cs) (không tái dùng `IpStack.Icmpv6`/`InternetChecksum`).
  - PackageReference (đặc thù): không có.
- **Được dùng bởi:** 7 project — driver **OpenVPN tap-mode** ([`Drivers.OpenVpn`](../TqkLibrary.VpnClient.Drivers.OpenVpn), V2.h), **SoftEther** ([`Drivers.SoftEther`](../TqkLibrary.VpnClient.Drivers.SoftEther), V4.c), **Vtun** ([`Drivers.Vtun`](../TqkLibrary.VpnClient.Drivers.Vtun) — `VirtualHost`+`ArpResolver`+`MacAddress`), **ZeroTier** ([`Drivers.ZeroTier`](../TqkLibrary.VpnClient.Drivers.ZeroTier) — `VirtualHost`+`ArpResolver`+`EthernetFrame`+`MacAddress`), **n2n** ([`Drivers.N2n`](../TqkLibrary.VpnClient.Drivers.N2n) — `VirtualHost`+`ArpResolver`+`MacAddress`), **VXLAN** ([`Drivers.Vxlan`](../TqkLibrary.VpnClient.Drivers.Vxlan) — VXLAN RFC 7348 L2-over-UDP, tái dùng `VirtualHost`+`ArpResolver`+`IEthernetChannel`), và **PPP** ([`Ppp`](../TqkLibrary.VpnClient.Ppp) — `Ipv6AddressConfigurator`+`NdiscResolver`+`EthernetFrame`+`MacAddress` cho IPv6 trên PPP). OpenVPN tap + SoftEther hiện bind **1** `VirtualHost` + `ArpResolver` (+ `DhcpV4Configurator` cho SoftEther) lên một `IEthernetChannel` **uplink VPN thật** (point-to-point tới server) → `IPacketChannel` mà `TcpIpStack` dùng. **`EthernetAdapter` (L2.7) nay ráp sẵn multi-host** (`EthernetSwitch` + N×`VirtualHost` + resolver/configurator + backpressure) — `AddHost(mac, build)` thực hiện đúng chuỗi compose + nối seam mà 2 driver đang làm thủ công (`host.InboundNonIpFrame += arp.HandleInboundFrame`; `host.InboundIpPacket += dhcp/ndisc.HandleInboundFrame`); **L2.8** phơi `MultiHostSession` (N station = N `IVpnSession`) trên `EthernetAdapter` + wire năng lực driver (OpenVPN tap ⇒ `L2Ethernet`/`L2BroadcastDomain`, SoftEther ⇒ `SupportsMultiHost`); **data-plane đa-host nay đã có** — `EthernetSwitch.ConnectUplink`/`EthernetAdapter.ConnectUplink` gắn uplink VPN (`SoftEtherEthernetChannel`/`OpenVpnTapChannel`) vào switch như một port; 2 driver bật qua opt-in `multiHost`. `EthernetSwitch` (L2.1) + `VirtualHost` (L2.2) + `ArpResolver` (L2.3) + `NdiscResolver` (L2.4) + `DhcpV4Configurator` (L2.5) + `Ipv6AddressConfigurator` (L2.6) đều ở src và đều được `EthernetAdapter` compose.

## API / cách dùng

```csharp
using TqkLibrary.VpnClient.Ethernet;

// 1) MAC: parse / format / cờ
MacAddress dst = MacAddress.Broadcast;                 // ff:ff:ff:ff:ff:ff
MacAddress src = MacAddress.Parse("02:00:00:00:00:01");
bool bcast = dst.IsBroadcast;                           // true
bool v6mc  = MacAddress.Parse("33:33:00:00:00:01").IsIpv6Multicast; // true

// 2) Dựng khung Ethernet II bọc một gói IP
byte[] frame = EthernetFrame.Build(dst, src, EthernetFrame.EtherTypeIpv4, ipPacket);

// 3) Đọc khung nhận về
MacAddress to    = EthernetFrame.Destination(frame);
ushort etherType = EthernetFrame.EtherType(frame);
ReadOnlyMemory<byte> payload = EthernetFrame.Payload(frame);

// 4) Switch học MAC: mỗi host nhận một IEthernetChannel làm port
await using var sw = new EthernetSwitch();
IEthernetChannel portA = sw.ConnectHost(MacAddress.Parse("02:00:00:00:00:0a"));
IEthernetChannel portB = sw.ConnectHost(MacAddress.Parse("02:00:00:00:00:0b"));
portB.InboundFrame += f => { /* host B nhận khung switch forward tới */ };
await portA.WriteFrameAsync(frame);   // switch học src của A, rồi forward/flood theo dst

// 5) VirtualHost + ArpResolver: một "máy ảo" trên LAN — expose IPacketChannel cho TcpIpStack
MacAddress mac = MacAddress.Parse("02:00:00:00:00:0a");
IPAddress  ip  = IPAddress.Parse("10.0.0.10");
IEthernetChannel port = sw.ConnectHost(mac);
await using var arp  = new ArpResolver(mac, ip, port);     // INeighborResolver IPv4 (RFC 826); gửi ARP qua port
await using var host = new VirtualHost(mac, port, arp);
host.InboundNonIpFrame += arp.HandleInboundFrame;          // nối seam: khung ARP inbound → resolver
var stack = new TcpIpStack(host, ip);   // stack chỉ thấy IPacketChannel — không biết Ethernet/ARP
// stack.ConnectAsync(...) → VirtualHost bọc Ethernet + ArpResolver resolve dest-MAC (ARP thật), đẩy qua switch

// 6) DhcpV4Configurator: xin IP/DNS/route từ DHCP server (SecureNAT / server-bridge) — trả TunnelConfig
await using var dhcp = new DhcpV4Configurator(mac, port);  // IAddressConfigurator (RFC 2131); gửi DHCP qua port
host.InboundIpPacket += dhcp.HandleInboundFrame;           // nối seam: reply DHCP (UDP/IPv4 67→68) → configurator
TunnelConfig config = await dhcp.ConfigureAsync();         // DISCOVER → OFFER → REQUEST → ACK
// config.AssignedAddress / config.PrefixLength (từ subnet-mask) / config.Routes (default-route) / config.DnsServers

// 7) EthernetAdapter (L2.7): ráp switch + N host — adapter tự compose VirtualHost + nối seam (không tự tay như 5/6)
await using var adapter = new EthernetAdapter();           // sở hữu một EthernetSwitch in-memory
EthernetAdapter.EthernetHostHandle h = adapter.AddHost(mac, p =>
{
    var arp2 = new ArpResolver(mac, ip, p);                 // resolver chia chung port p adapter vừa connect
    var dhcp2 = new DhcpV4Configurator(mac, p);
    return new EthernetHostSpec(arp2)                       // adapter nối NonIp→ARP, Ip→DHCP rồi bọc backpressure
    {
        Configurator = dhcp2,
        NonIpFrameHandler = arp2.HandleInboundFrame,
        IpPacketHandler = dhcp2.HandleInboundFrame,
    };
});
TunnelConfig leased = await h.ConfigureAsync();             // chạy DHCP/SLAAC của host
var stack2 = new TcpIpStack(h.Channel, leased.AssignedAddress!);  // stack bind IPacketChannel backpressure của host
// thêm host khác: adapter.AddHost(mac2, ...) — mỗi host một MAC + IPacketChannel độc lập trên cùng switch
```

## Luồng nội bộ

- **`MacAddress`** ([MacAddress.cs:13](MacAddress.cs#L13)): sáu octet đóng gói big-endian vào low-48-bit của một `ulong` (octet 0 = byte cao nhất). Equality/hash so trên `ulong` nên rẻ → dùng trực tiếp làm khoá `Dictionary` cho FDB switch (L2.1). `IsMulticast` đọc bit I/G của octet đầu ([:34](MacAddress.cs#L34)); `IsIpv6Multicast` kiểm prefix `33:33` ([:37](MacAddress.cs#L37)). `TryParse` ([:76](MacAddress.cs#L76)) chấp nhận phân tách `:` hoặc `-`, đòi đúng 6 octet hex 2 chữ số.
- **`EthernetFrame.Build`** ([EthernetFrame.cs:23](EthernetFrame.cs#L23)): cấp `byte[14 + payload]`, ghi dst/src MAC qua `MacAddress.CopyTo`, EtherType big-endian (byte 12-13), rồi copy payload — đối xứng `Ipv4.Build`. Readers ([:35-44](EthernetFrame.cs#L35-L44)) đọc trực tiếp từ `ReadOnlySpan<byte>`/`ReadOnlyMemory<byte>` không cấp phát (trừ `MacAddress.FromBytes`).
- **`EthernetSwitch.OnIngress`** ([EthernetSwitch.cs:75](EthernetSwitch.cs#L75)): mỗi frame host gửi vào (`Port.WriteFrameAsync`) được switch (a) **học** `_fdb[srcMAC] = ingressPort` (MAC-move tự ghi đè), rồi (b) **chọn đích dưới `_sync`**: dst là multicast (gồm broadcast) **hoặc** unicast chưa-học → gom danh sách flood (mọi port ≠ ingress); unicast đã-học & port ≠ ingress → đúng 1 port; unicast đã-học & port == ingress → drop (không phản xạ). (c) **Deliver ngoài lock** (`Port.Deliver` → raise `InboundFrame`) để handler host có thể ghi lại đồng bộ mà không deadlock. `RemovePort` ([:111](EthernetSwitch.cs#L111)) khi disconnect gỡ port khỏi `_ports` + xoá mọi FDB entry trỏ tới nó (đích cũ thành unknown → flood lại).
- **`VirtualHost`** ([VirtualHost.cs:23](VirtualHost.cs#L23)): object **là** `IPacketChannel` mà `TcpIpStack` bind, đồng thời giữ MAC + port switch + resolver. **Egress** ([`WriteIpPacketAsync` @ :69](VirtualHost.cs#L69)): đọc version-nibble (`packet[0]>>4`) chọn EtherType + trích IP đích từ offset cố định (v4 @16, v6 @24 — không ref `IpStack` để giữ no-horizontal-dep), `await resolver.ResolveAsync(dstIp)` lấy MAC (on-link: next-hop = IP đích; `null` → drop), `EthernetFrame.Build` → `port.WriteFrameAsync`. **Ingress** ([`OnInboundFrame` @ :106](VirtualHost.cs#L106)) đăng ký `port.InboundFrame`: khung < 14 byte → drop; EtherType IPv4/IPv6 → `InboundIpPacket?.Invoke(Payload)` (slice zero-copy, valid trong handler vì switch raise đồng bộ); khác (ARP) → `InboundNonIpFrame?.Invoke(frame)` (seam cho L2.3). `Mtu=link−14` tính 1 lần trong ctor; `DisposeAsync` unsubscribe + null event + `port.DisposeAsync()` (gỡ khỏi switch).
- **`ArpResolver`** ([ArpResolver.cs:26](ArpResolver.cs#L26)): hiện thực `INeighborResolver` cho IPv4, giữ MAC + IPv4 của ta + chia chung `port` switch (gửi ARP) + 2 `Dictionary` (cache IP→MAC có `DateTime` expiry; pending TCS). **Egress** ([`ResolveAsync` @ :73](ArpResolver.cs#L73)): không phải IPv4 → `null` (NDISC v6 để L2.4); dưới lock cache-hit chưa hết hạn → trả ngay, miss → tạo/lấy pending TCS (`RunContinuationsAsynchronously`, owner-coalesce các resolve trùng IP). Ngoài lock owner chạy ≤`MaxAttempts` vòng: broadcast ARP request ([`SendRequestAsync` @ :176](ArpResolver.cs#L176)) rồi `await` reply có timeout ([`AwaitReplyAsync` @ :195](ArpResolver.cs#L195), `Task.WhenAny` + linked-CTS, netstandard2.0 thiếu `Task.WaitAsync`); hết attempts → `TrySetResult(null)`. **Ingress** ([`HandleInboundFrame` @ :132](ArpResolver.cs#L132), wired `host.InboundNonIpFrame`): guard len/EtherType/`IsIpv4OverEthernet`; dưới lock học `_cache[senderIp]=senderMac` (mọi gói ARP — RFC 826 merge) + complete/remove pending của `senderIp`; nếu `op==Request && targetIp==ta` → đánh dấu reply. Ngoài lock complete TCS + (nếu reply) gửi unicast reply qua `port` — **gửi/complete ngoài lock** (mẫu switch) tránh deadlock khi round-trip nội-tuyến trên fabric đồng bộ. `DisposeAsync` drain pending → `null`, clear cache, **không** dispose `port` (VirtualHost sở hữu).
- **`NdiscResolver`** ([NdiscResolver.cs:31](NdiscResolver.cs#L31)): hiện thực `INeighborResolver` cho IPv6, cấu trúc mirror `ArpResolver` (MAC + IPv6 + `port` + cache IPv6→MAC + pending TCS), thêm slot DAD (`_dadDefended`/`_dadTarget`) + RA-cache (`_lastRa`). **Egress** ([`ResolveAsync` @ :74](NdiscResolver.cs#L74)): không phải IPv6 → `null` (ARP v4 để L2.3); cache-hit chưa hết hạn trả ngay, miss → pending TCS owner-coalesce; ngoài lock chạy ≤`MaxAttempts` vòng gửi **NS tới solicited-node multicast** ([`SendSolicitationAsync`](NdiscResolver.cs#L337), dst MAC `33:33:ff:..` qua [`Icmpv6Ndisc.MulticastMac`](Icmpv6Ndisc.cs#L108)) rồi `await` NA có timeout (`AwaitReplyAsync`, giống ARP). **Ingress** ([`HandleInboundFrame` @ :182](NdiscResolver.cs#L182), wired **cả** `InboundIpPacket` lẫn `InboundNonIpFrame`): nhận khung Ethernet hoặc gói IPv6 trần, guard version-6/next-header-58/`IsNdisc`; **NA** → học Target-LLA vào cache + complete pending + (nếu target == địa chỉ DAD của ta) đánh dấu defended; **NS** → học Source-LLA (nếu có) + nếu target == ta gửi **NA solicited** (unicast về source, hoặc all-nodes nếu DAD probe từ `::`); **RA** → parse gateway (source) + Source-LLA + Prefix-Info vào [`RouterAdvertisementInfo`](Models/RouterAdvertisementInfo.cs#L12) set `_lastRa` + raise `RouterAdvertisementReceived`. **DAD** ([`PerformDuplicateAddressDetectionAsync` @ :134](NdiscResolver.cs#L134)): gửi `DadTransmits` lần NS **từ `::`** (omit Source-LLA) tới solicited-node multicast của địa chỉ ta, mỗi lần chờ `DadTimeout` — có NA defend ⇒ `false` (trùng), im hết ⇒ `true` (unique). Gửi/complete **ngoài lock** (mẫu switch). `DisposeAsync` drain pending → `null` + giải phóng DAD (coi như unique), **không** dispose `port`.
- **`DhcpV4Configurator`** ([DhcpV4Configurator.cs:29](DhcpV4Configurator.cs#L29)): hiện thực `IAddressConfigurator` cho IPv4, giữ MAC + `port` switch + 1 pending TCS + `xid` hiện hành. **`ConfigureAsync`** ([:55](DhcpV4Configurator.cs#L55)): sinh `xid` ngẫu nhiên (RNG) → `ExchangeAsync(DISCOVER, …, OFFER)` lấy yiaddr + server-id (option 54, fallback siaddr) → `ExchangeAsync(REQUEST[option 50=offered, 54=server], …, ACK)` → `BuildTunnelConfig`. **`ExchangeAsync`** ([:78](DhcpV4Configurator.cs#L78)): đặt pending TCS + `_xid`, `BroadcastFrame` (build DHCP → `DhcpV4Packet.BuildUdpIpv4` 0.0.0.0:68→255.255.255.255:67 → `EthernetFrame.Build` dst broadcast) qua `port.WriteFrameAsync`, `await` reply có timeout (`AwaitReplyAsync`, `Task.WhenAny`+linked-CTS như ARP/NDISC); timeout → retransmit ≤`MaxAttempts`; **NAK** ⇒ ném `InvalidOperationException`; type khớp ⇒ trả; hết attempts ⇒ ném. **Ingress** ([`HandleInboundFrame` @ :120](DhcpV4Configurator.cs#L120), wired **cả** `InboundIpPacket` lẫn `InboundNonIpFrame`): nhận khung Ethernet hoặc gói IPv4 trần → `DhcpV4Packet.TryReadUdpIpv4` (lọc UDP dst port 68) → dưới lock kiểm `IsReplyFor(xid)` (BOOTREPLY + xid khớp + magic-cookie) → complete pending TCS. `BuildTunnelConfig` ([:200](DhcpV4Configurator.cs#L200)): IP=yiaddr, prefix=`MaskToPrefix(option 1)` (fallback /32), default-route `0.0.0.0/0 <router>` từ option 3, DNS từ option 6, MTU theo options. Complete **ngoài lock** (`RunContinuationsAsynchronously`); `DisposeAsync` cancel pending. DHCP ride **trong IPv4 broadcast** (không EtherType riêng), nên composer nối `host.InboundIpPacket += dhcp.HandleInboundFrame` (mirror NDISC). Tự dựng IPv4+UDP header trong `DhcpV4Packet` (checksum RFC 768/791) — **không** ref `IpStack` (giữ no-horizontal-dep, mirror `Icmpv6Ndisc`).
- **`Ipv6AddressConfigurator`** ([Ipv6AddressConfigurator.cs:34](Ipv6AddressConfigurator.cs#L34)): hiện thực `IAddressConfigurator` cho IPv6, giữ MAC + link-local + `port` switch + ref `NdiscResolver` (L2.4) + DUID-LL client + 1 pending DHCPv6 TCS + 1 RA-waiter TCS + `xid`. **`ConfigureAsync`** ([:71](Ipv6AddressConfigurator.cs#L71)): `AcquireRouterAdvertisementAsync` (đọc `NdiscResolver.LastRouterAdvertisement`, nếu chưa có thì đăng ký RA-waiter + gửi **RS** tới `ff02::2` rồi chờ event `RouterAdvertisementReceived`, retry ≤`RouterSolicitationAttempts`) → từ RA: nếu **prefix autonomous /64** + lifetime≠0 ⇒ `TryFormSlaacAddress` (ghép prefix + iid EUI-64/opaque qua `SlaacAddress`) set `AssignedAddressV6`; thêm default-route `::/0 <router>`; nếu **M** | `ForceDhcp` | không-SLAAC ⇒ `RunDhcpv6Async` (SOLICIT→ADVERTISE lấy server-id + offered addr, REQUEST[server-id + IA-Address echo]→REPLY) lấy địa chỉ (nếu chưa có SLAAC) + DNS; cờ **O** chỉ kéo DNS. Không địa chỉ nào ⇒ ném. **`ExchangeAsync`** ([:197](Ipv6AddressConfigurator.cs#L197)): đặt pending TCS + `xid`, `MulticastFrame` (build DHCPv6 → `Dhcpv6Packet.BuildUdpIpv6` link-local:546→ff02::1:2:547 → MAC `33:33:00:01:00:02`) qua `port.WriteFrameAsync`, `await` reply có timeout (`AwaitAsync`, `Task.WhenAny`+linked-CTS như ARP/DHCPv4); timeout ⇒ retransmit ≤`DhcpMaxAttempts`. **Ingress** ([`HandleInboundFrame`](Ipv6AddressConfigurator.cs#L236), wired **cả** `InboundIpPacket` lẫn `InboundNonIpFrame`): `Dhcpv6Packet.TryReadUdpIpv6` (lọc UDP dst port 546) → dưới lock kiểm `xid` khớp → complete pending. RA do `NdiscResolver` parse (không ở đây). Complete **ngoài lock**; `DisposeAsync` unsubscribe RA event + cancel pending + RA-waiter. Tự dựng IPv6+UDP header trong `Dhcpv6Packet` (checksum pseudo-header RFC 8200) — **không** ref `IpStack` (mirror `Icmpv6Ndisc`/`DhcpV4Packet`).

## Trạng thái & ghi chú

- **Đã hiện thực (L2.0):** `MacAddress`, `EthernetFrame`, và 2 slot interface (`INeighborResolver`/`IAddressConfigurator` ở Abstractions).
- **Đã hiện thực (L2.1):** `EthernetSwitch` (+ nested `Port`) — switch học MAC in-memory, là src type thật đầu tiên.
- **Đã hiện thực (L2.2):** [`VirtualHost`](VirtualHost.cs#L23) (`: IPacketChannel`) — cầu nối L2↔L3 expose `IPacketChannel` cho `TcpIpStack` (wrap/strip Ethernet, resolve dest-MAC qua `INeighborResolver`, MTU=link−14). Hiện thực **quy tắc vàng**: stack không bao giờ thấy Ethernet.
- **Đã hiện thực (L2.3):** [`ArpResolver`](ArpResolver.cs#L26) (`INeighborResolver` IPv4, RFC 826) + codec [`ArpPacket`](ArpPacket.cs#L12) — resolve next-hop IP→MAC khi egress (cache + broadcast request/reply + timeout/retry), học mọi gói ARP, trả lời request hỏi IP của ta. Nối khung ARP inbound qua seam `VirtualHost.InboundNonIpFrame` (`HandleInboundFrame`); chỉ phụ thuộc Abstractions + codec cùng project (không ref `VirtualHost` → không vòng).
- **Đã hiện thực (L2.4):** [`NdiscResolver`](NdiscResolver.cs#L31) (`INeighborResolver` IPv6, RFC 4861, mirror `ArpResolver`) + codec [`Icmpv6Ndisc`](Icmpv6Ndisc.cs#L21) + POCO [`RouterAdvertisementInfo`](Models/RouterAdvertisementInfo.cs#L12) — resolve IPv6→MAC qua NS-to-solicited-node/NA, học từ NA/NS-source-LLA, trả NA cho NS hỏi ta; **DAD** (NS từ `::` phát hiện trùng); **parse RA** lấy gateway+prefix (hand-off cho SLAAC L2.6/P1.1). NDISC ride trong IPv6 nên `HandleInboundFrame` nối **cả** `VirtualHost.InboundIpPacket` lẫn `InboundNonIpFrame`. Checksum pseudo-header tính nội bộ trong `Icmpv6Ndisc` (không ref `IpStack`).
- **Đã hiện thực (L2.5):** [`DhcpV4Configurator`](DhcpV4Configurator.cs#L29) (`IAddressConfigurator` IPv4, RFC 2131) + codec [`DhcpV4Packet`](DhcpV4Packet.cs#L17) (BOOTP + UDP/IPv4 framing) + [`DhcpV4Options`](DhcpV4Options.cs#L15) (magic-cookie + TLV) — handshake **DISCOVER→OFFER→REQUEST→ACK** qua Ethernet broadcast UDP 68→67, `xid` tương quan + retransmit, trả `TunnelConfig` (IP=yiaddr, prefix từ subnet-mask, default-route từ router, DNS, MTU); NAK/timeout ⇒ ném. Tự dựng IPv4+UDP header (RFC 768/791) — **không** ref `IpStack` (mirror `Icmpv6Ndisc`). DHCP ride trong IPv4 broadcast nên `HandleInboundFrame` nối `VirtualHost.InboundIpPacket` (+ `InboundNonIpFrame`). **Mở đường V.4 SoftEther SecureNAT + OpenVPN tap pure-DHCP-bridge (V2.h).**
- **Đã hiện thực (L2.6):** [`Ipv6AddressConfigurator`](Ipv6AddressConfigurator.cs#L34) (`IAddressConfigurator` IPv6, RFC 4862 + 8415) + codec [`Dhcpv6Packet`](Dhcpv6Packet.cs#L15) (message + UDP/IPv6 framing 546↔547 tới `ff02::1:2`) + [`Dhcpv6Options`](Dhcpv6Options.cs#L14) (DUID-LL + IA_NA/IA-Address + Option-Request + Status + DNS) + helper [`SlaacAddress`](Helpers/SlaacAddress.cs#L13) (EUI-64/opaque iid + combine) + enum [`SlaacInterfaceIdentifierMode`](Enums/SlaacInterfaceIdentifierMode.cs#L7). - **⚠️ "LAN ảo" này hiện là fabric LOCAL in-process, CHƯA nối VPN.** Mục tiêu thiết kê ([03](../../.docs/03-multihost-l2-vs-l3.md)): LAN ảo = **broadcast domain do L2 VPN server cấp** (SoftEther bridge / OpenVPN-tap) — client mở **1 kết nối VPN** nhận khung Ethernet thật, rồi đặt **N `VirtualHost`** (mỗi MAC+IP riêng, ARP/DHCP do **server** trả lời) lên đó. **OpenVPN tap-mode (V2.h) đã** sinh `IEthernetChannel` thật từ kết nối VPN (uplink **point-to-point** tới server — `OpenVpnTapChannel`) + bind **1** `VirtualHost` → `IPacketChannel` (server trả lời ARP, IP từ ifconfig). **Data-plane đa-host nay đã có**: `EthernetSwitch.ConnectUplink`/`EthernetAdapter.ConnectUplink` cắm uplink VPN vào switch như một port — SoftEther + OpenVPN-tap (opt-in `multiHost`) gắn `SoftEtherEthernetChannel`/`OpenVpnTapChannel` làm uplink, các `VirtualHost` station ARP/DHCP **xuyên uplink ra server thật** (server cấp cả broadcast domain + DHCP từng station), `OpenSessionAsync` thêm station. - **Cố ý lược bỏ codec** (tối giản, ghi rõ trong [EthernetFrame.cs:3-6](EthernetFrame.cs#L3-L6)):
  - **VLAN 802.1Q tag** (4 byte chèn sau src MAC) — chưa cần cho LAN phẳng.
  - **FCS** (4 byte CRC cuối khung) — NIC/driver phần cứng tự thêm; fabric phần mềm không kiểm.
  - **Padding tối thiểu 60 byte** — chỉ cần trên dây vật lý; switch phần mềm bỏ qua.
- **Cố ý lược bỏ switch** ([EthernetSwitch.cs:12-15](EthernetSwitch.cs#L12-L15)): **FDB aging/timeout** (entry sống tới khi disconnect/MAC-move), **STP** (giả định topo không vòng), **VLAN**, **IGMP/MLD snooping** (mọi multicast đều flood — chưa lọc nhóm).
- **Cố ý lược bỏ VirtualHost** ([VirtualHost.cs:17-21](VirtualHost.cs#L17-L21)): **routing/gateway** (next-hop = IP đích, giả định on-link — chưa chọn gateway cho đích off-link); **read-loop/Pipe per-host** (design `09` đặt mục tiêu 1 read-loop + backpressure, nhưng hiện event-driven callback như phần data plane còn lại — để mục P2 hiệu năng); **không** ref project `IpStack` (đọc header L3 bằng offset cố định để tránh phụ thuộc ngang).
- **Cố ý lược bỏ ArpResolver** (L2.3 tối giản): **proxy-ARP** (trả lời thay host khác) + **gratuitous ARP announce** (tự quảng bá IP khi lên link); **cache aging-sweep chủ động** (hiện chỉ **lazy-expire** khi resolve, không có timer quét nền); **routing/gateway** (next-hop = IP đích như VirtualHost). NDISC (IPv6) là **L2.4 = `NdiscResolver`** riêng — ARP chỉ IPv4, next-hop v6 → `null`. *(DHCP L2.5 gửi broadcast L2 thẳng `ff:ff:ff:ff:ff:ff` trong `DhcpV4Configurator`, không cần ARP map IP-broadcast→MAC.)*
- **Cố ý lược bỏ NdiscResolver** (L2.4 tối giản): **gửi Router Solicitation** (resolver host chỉ parse RA nhận được — RS nay **do `Ipv6AddressConfigurator` L2.6 gửi**, không phải resolver); **NUD** (Neighbor Unreachability Detection state machine REACHABLE/STALE/PROBE — hiện chỉ TTL lazy-expire như ARP); **on-link prefix / route table đầy đủ** (next-hop = IP đích on-link). NDISC chỉ IPv6, next-hop v4 → `null`. *(SLAAC tạo địa chỉ + DHCPv6 nay là **L2.6 = `Ipv6AddressConfigurator`**, tiêu thụ `RouterAdvertisementInfo` của L2.4.)*
- **Cố ý lược bỏ Ipv6AddressConfigurator** (L2.6 tối giản): **DAD trên địa chỉ SLAAC mới** (giả định unique trên fabric point-to-point — `NdiscResolver.PerformDuplicateAddressDetectionAsync` có sẵn nếu cần); **lease renew/rebind (T1/T2) + Reconfigure + RELEASE/DECLINE** DHCPv6 (acquire 1 lần, không gia hạn nền); **Rapid Commit (2-message)** + **Prefix Delegation (IA_PD)** + **temporary addresses (IA_TA)** + **multiple IA / IA-Address**; **xử lý nhiều prefix RA** (chỉ lấy prefix đầu); **on-link route từ prefix L-flag**; **xác minh checksum reply** (tin server trên fabric như phần còn lại của L2). Privacy-extension RFC 8981 (địa chỉ tạm random đổi định kỳ) chưa làm — `StableOpaque` (RFC 7217) là phương án ổn định không lộ MAC.
- **Cố ý lược bỏ DhcpV4Configurator** (L2.5 tối giản): **lease renew/rebind/expiry timer** (T1/T2, RFC 2131 §4.4.5 — hiện chỉ acquire 1 lần, không gia hạn nền; lease-time parse ra nhưng chưa lập lịch); **DECLINE sau DAD-xung-đột** + **RELEASE khi teardown**; **INFORM** (đã có IP, chỉ xin config); **option 82 relay-agent** + **giaddr relay** (chỉ exchange trực tiếp); **client-FQDN/option-overload**; **xác minh checksum reply** (tin server trên fabric point-to-point/in-memory như phần còn lại của tầng L2). DHCP chỉ IPv4 — IPv6 là SLAAC/DHCPv6 (**L2.6**).
