# TqkLibrary.VpnClient.Drivers.L2tpIpsec

> Driver VPN **L2TP/IPsec**: IKEv1 PSK (Main + Quick Mode) qua NAT-T, ESP transport mode, L2TPv2, PPP/MS-CHAPv2. Đây là nơi lắp ráp toàn bộ stack giao thức thành một `IVpnConnection` hoàn chỉnh.

## Mục đích

Project này là **một driver của tầng DRIVER** — điểm điều phối control plane cho giao thức L2TP/IPsec. Nó lấy các khối giao thức rời rạc (IKE, ESP, L2TP, PPP, NAT-T transport) và **lắp ráp chúng thành một đường hầm hoàn chỉnh**, rồi bọc lại sau interface trung lập `IVpnConnection` để tầng `TqkLibrary.VpnClient` ở trên tiêu thụ mà không cần biết giao thức cụ thể.

Driver L2TP/IPsec ([L2tpIpsecDriver.cs:12](L2tpIpsecDriver.cs#L12)) gồm: IKEv1 PSK (Main Mode + Quick Mode) qua NAT-T (UDP 500→4500), ESP transport mode làm data plane, L2TPv2 trên UDP/1701, và PPP/MS-CHAPv2 để nhận IP. Kèm **vòng đời đầy đủ**: keepalive (L2TP HELLO + IKE DPD), rekey ESP CHILD SA **và ISAKMP SA in-place** (make-before-break), teardown sạch (IKE Delete + L2TP CDN/StopCCN), và auto-reconnect (exponential backoff) sau một facade kênh ổn định.

**Carrier ESP 2 nhánh:** mặc định **ESP-in-UDP/4500** (NAT-T, no-admin). Khi `HonestFirst` phát hiện gateway **no-NAT** và app cấp `IRawIpTransportFactory` (đã elevate), driver chở **native ESP trên IP proto-50** qua [`Transport.RawIp`](../TqkLibrary.VpnClient.Transport.RawIp) — IKE vẫn ở UDP/500, **gói ESP/DPD/rekey y hệt, chỉ khác carrier ngoài cùng** (roadmap **P0.8c**, opt-in; null = no-admin như cũ).

Vấn đề được giải quyết: tách phần "biết cách nói chuyện với gateway L2TP/IPsec" ra khỏi phần "biết cách dùng đường hầm" (sockets/IP stack ở trên), nhờ đảo ngược phụ thuộc qua `IVpnProtocolDriver`/`IVpnConnection` trong `Abstractions`.

> Driver MS-SSTP nằm ở project anh em [TqkLibrary.VpnClient.Drivers.Sstp](../TqkLibrary.VpnClient.Drivers.Sstp). Trước đây hai driver chung một project `TqkLibrary.VpnClient.Drivers` (subfolder `L2tpIpsec/`+`Sstp/`); đã tách thành 2 project, file phẳng ở gốc, namespace giữ nguyên.

## API / cách dùng

Điểm vào công khai là driver `L2tpIpsecDriver` (`IVpnProtocolDriver`). Thông thường tầng `TqkLibrary.VpnClient` (`VpnClientBuilder.UseL2tpIpsec()`) đăng ký driver và gọi `ConnectAsync`; có thể dùng trực tiếp:

```csharp
// L2TP/IPsec — auto-reconnect bật mặc định
var driver = new L2tpIpsecDriver(); // hoặc new L2tpIpsecDriver(new L2tpIpsecReconnectOptions { Enabled = false })
IVpnConnection conn = await driver.ConnectAsync(
    new VpnEndpoint("vpn.example.com", 0),
    new VpnCredentials { Username = "user", Password = "pass", PreSharedKey = Encoding.ASCII.GetBytes("vpn") }); // PSK bắt buộc (P0.4)

IVpnSession session = conn.Sessions[0];
IPacketChannel channel = session.PacketChannel;   // kênh L3 ổn định, sống qua reconnect
IPAddress myIp = session.Config.AssignedAddress;
await conn.DisposeAsync();                          // teardown sạch: IKE Delete + L2TP CDN/StopCCN
```

**PSK bắt buộc (P0.4):** driver **không** nhét PSK mặc định — `VpnCredentials.PreSharedKey` null/rỗng ⇒ ném `ArgumentException` ngay trước khi mở mạng ([L2tpIpsecDriver.cs:60-64](L2tpIpsecDriver.cs#L60-L64)). Default credential đặc thù VPN Gate (group PSK `"vpn"`) **không** thuộc lib chung; caller (vd demo) tự cấp.

Class hạ tầng `L2tpIpsecConnection` cũng public nếu cần điều khiển chi tiết (sự kiện `StateChanged`/`Reconnected`, `DisconnectAsync`...). Lỗi kết nối ném typed exception (`VpnAuthenticationException` / `VpnServerRejectedException` / `VpnNetworkTimeoutException` — đều kế thừa `VpnConnectionException`).

**Timeout cấu hình được:** truyền `L2tpIpsecTimeoutOptions` (ctor driver/connection hoặc `VpnClientBuilder.UseL2tpIpsec(reconnect, timeout)`) để chỉnh số lần/khoảng retransmit IKE (`IkeMaxAttempts`/`IkeRetransmitInterval`, mặc định 5×2.5s) và L2TP control channel (`L2tpMaxRetransmits`/`L2tpRetransmitInterval`, mặc định 8×1s). IKE hết số lần ⇒ `VpnNetworkTimeoutException` (**phân loại theo cổng — P0.8a**: im lặng sau float UDP/4500 ⇒ thông điệp nghi forced-NAT-T); gateway trả NOTIFY lỗi handshake ⇒ `VpnServerRejectedException`; L2TP hết cap ⇒ link-loss → reconnect. **Chế độ NAT-T:** `natTraversalMode` (ctor driver/connection) — `ForcedNatT` (mặc định, ép float 4500) hoặc `HonestFirst` (**P0.8b** — bind cổng 500 thật + NAT-D trung thực, tự fallback forced khi không bind được/no-NAT). **Native ESP proto-50 (P0.8c, opt-in):** truyền `IRawIpTransportFactory` (`new L2tpIpsecDriver(rawIpFactory: …)` hoặc `VpnClientBuilder.UseL2tpIpsec(new RawIpTransportFactory())` — mode mặc định `HonestFirst`); khi gateway no-NAT và factory khả dụng (đã elevate), ESP chạy native trên IP proto-50 thay vì float 4500. **null = no-admin như cũ.**

## Luồng nội bộ

> **Supervisor F.6**: `L2tpIpsecConnection : ReconnectingVpnConnection<L2tpIpsecConnectionState>` (Drivers.Core). Base sở hữu facade `SwappablePacketChannel`, lifetime cancel, backoff-jitter, `StateChanged`, `OnLinkLost`/`ReconnectLoopAsync`, teardown chung, clock; driver override `EstablishAsync`/`CleanupAttemptResourcesAsync`/`StopAttemptLoop` + ánh xạ 4 state + `OnReconnected`/`DisconnectAsync`. Rekey Phase1/Phase2 + multi-session + DPD/HELLO + swap SA giữ trong driver trên timer riêng — **ngoài** supervisor (làm tươi SA tại chỗ, không re-establish).

### Handshake (trong `EstablishAsync` — clean-slate, dùng chung cho connect đầu và mọi reconnect)

1. Resolve host qua `IHostResolver` theo `AddressFamilyPreference` (P1.2 — ưu tiên họ IP config, fallback họ còn lại; thay filter cứng IPv4 cũ) → `StartAttemptChannel` tạo `NatTraversalChannel` (UDP/500, **bind đúng họ địa chỉ server**) + chạy `ReceiveLoopAsync` ghép kênh IKE/ESP — [L2tpIpsecConnection.cs:400](L2tpIpsecConnection.cs#L400). Trước đó `CleanupAttemptResourcesAsync` (override hook base) dọn attempt cũ: cancel vòng đọc, **null+dispose `_nativeEsp`** (nếu có) rồi null `_natt` + **await** `DisposeAsync` (socket cũ đóng hẳn trước khi mở socket mới) — [L2tpIpsecConnection.cs:425](L2tpIpsecConnection.cs#L425); vòng đọc còn **guard sau mỗi `ReceiveAsync`** (re-check token + identity channel) để gói stale đang in-flight không lọt vào waiter của attempt mới — [L2tpIpsecConnection.cs:541](L2tpIpsecConnection.cs#L541).
2. **IKEv1 Phase 1 (MM1–MM4) + quyết định carrier ESP** trong `BringUpPhase1Async` theo `L2tpIpsecNatTraversalMode` (mặc định `ForcedNatT`) — [L2tpIpsecConnection.cs:328-396](L2tpIpsecConnection.cs#L328-L396):
   - **ForcedNatT** (đường live đang chạy): cổng nguồn ephemeral, NAT-D **khai man** nguồn = `(Any, 500)` ⇒ ép gateway thấy NAT ⇒ **luôn** `SwitchToNatTPort()` sang 4500 (ESP-in-UDP).
   - **HonestFirst** (opt-in **P0.8b**): thử-bind cổng nguồn **500 thật** (bind fail, vd Windows IKEEXT ⇒ fallback forced), rồi tính `localIp = natt.GetLocalAddress()` và tạo `IkeV1Client(psk, localIp, serverIp)` — **honest identity = địa chỉ nguồn THẬT** làm Phase 1 / Quick Mode ID_ci (khác forced path cố ý khai `IPAddress.Any`); nếu gửi ID_ci = `0.0.0.0` trong khi gói tới từ địa chỉ thật, gateway (vd strongSwan) **tưởng có NAT** ⇒ cài SA UDP-encap (espinudp) ⇒ drop native proto-50 = `XfrmInStateMismatch` — đây là **bug đã sửa qua validate live P0.8c** ([`L2tpIpsecConnection.cs` :356-396](L2tpIpsecConnection.cs#L356-L396)). NAT-D **trung thực**, rồi `IkeV1Client.DetectNat` → hàm thuần [`L2tpIpsecNatStrategy.Decide`](L2tpIpsecNatStrategy.cs): **(a)** có NAT ⇒ float 4500; **(b)** no-NAT 2 phía **và** `IRawIpTransportFactory` khả dụng (đã elevate) ⇒ **native ESP (P0.8c)** — giữ IKE ở UDP/500, mở raw socket IP proto-50 bind đúng nguồn IKE; **(c)** còn lại ⇒ teardown rồi **fallback forced**.
3. **MM5/MM6** (đã mã hoá) trên cổng Phase 1 đã chốt, xác thực PSK qua `HASH_R`; sai PSK ⇒ `VpnAuthenticationException` — [L2tpIpsecConnection.cs:223-229](L2tpIpsecConnection.cs#L223-L229).
4. **Phase 2 (Quick Mode)** QM1–QM3 → khoá ESP CHILD SA + `NegotiatedEsp` (suite server chọn: AES-CBC mặc định, hoặc AES-GCM nếu gateway hỗ trợ); HASH(2) sai/không có SA ⇒ `VpnServerRejectedException` — [L2tpIpsecConnection.cs:233-237](L2tpIpsecConnection.cs#L233-L237). **Encapsulation-mode order (P0.8c):** mỗi cipher chào ở 2 mode `UDP-Encapsulated-Transport`(4) + `Transport`(2); responder chọn transform **đầu tiên** quyết SA install (Transport ⇒ native, UDP-encap ⇒ espinudp). Nhánh native bật [`IkeV1Client.PreferNativeTransport`](../TqkLibrary.VpnClient.Ipsec/Ike/V1/IkeV1Client.cs#L114) ⇒ [`IkeV1Proposals.Phase2`](../TqkLibrary.VpnClient.Ipsec/Ike/V1/IkeV1Proposals.cs#L66) xếp `Transport` TRƯỚC để server cài native SA dù no-NAT (forced path GIỮ UDP-encap trước). **Phân loại lỗi handshake (P0.8a)** ở [`ExchangeIkeAsync` :477](L2tpIpsecConnection.cs#L477): reply là NOTIFY lỗi (`IkeV1Client.TryReadRejectNotify`) ⇒ `VpnServerRejectedException` nêu mã; **no-response theo cổng** — im lặng sau float UDP/4500 ⇒ `VpnNetworkTimeoutException` chỉ rõ **nghi forced-NAT-T**, trên cổng 500 ⇒ "host unreachable/UDP blocked".

Dựng `EspSession` + chọn **sink ESP** (native `RawIpDatagramTransport.SendAsync` proto-50, hoặc `natt.SendEspAsync` ESP-in-UDP — **gói ESP y hệt**) → `IpsecL2tpTransport`; nếu native ⇒ `StartNativeEspLoop` chạy vòng đọc raw riêng (`_espActive`+identity-guard); `L2tpClient.ConnectAsync` (L2TP tunnel + session chính `PrimarySession`); `L2tpPppFrameChannel(l2tp.PrimarySession)` + `PppEngine` + `MsChapV2Authenticator` chạy LCP/Auth/IPCP (auth fail ⇒ `VpnAuthenticationException`) — [L2tpIpsecConnection.cs:240-288](L2tpIpsecConnection.cs#L240-L288).

### Keepalive · rekey · teardown · reconnect

- **Keepalive:** L2TP HELLO (60s) + IKE DPD R-U-THERE (20s, miss ≥3 ⇒ link lost) — [L2tpIpsecConnection.cs:601-660](L2tpIpsecConnection.cs#L601-L660). `StopAttemptLoop` (override — [L2tpIpsecConnection.cs:619](L2tpIpsecConnection.cs#L619)) dispose 5 timer (hello/dpd/rekey-Phase2/Phase1/drop) + tắt `_espActive`; cờ "tunnel up" do base quản (`IsRunning`).
- **Rekey Phase 2 (make-before-break) — 2 trigger:** (a) timer ~90% lifetime (3600s); (b) **sequence-exhaustion** — sequence ESP outbound chạm high-watermark ~75%×2³² thì `RekeyNeeded` kích rekey **trước khi** wrap (RFC 4303 §3.3.3, tránh `OverflowException`). Cả hai vào `RekeyPhase2Async` (guard `_rekeyInProgress`) → Quick Mode mới → `SwapSession` (gửi trên SA mới ngay + re-arm watermark, giữ SA cũ để nhận trong grace 10s rồi `DropPreviousInbound`) — [L2tpIpsecConnection.cs:701-750](L2tpIpsecConnection.cs#L701-L750) · [EspDataPlane.cs:40-98](../TqkLibrary.VpnClient.Ipsec/Esp/EspDataPlane.cs#L40-L98) (scaffolding swap/watermark dùng chung, tách khỏi `IpsecL2tpTransport`).
- **Rekey Phase 1 in-place (make-before-break, P1.3):** timer ~90% Phase 1 (8h) vào `RekeyPhase1Async` — dựng **ISAKMP SA mới hoàn toàn** (Main Mode đầy đủ, cookie/DH/SKEYID\* mới) trên **cùng kênh handshake đã chốt** (forced NAT-T = UDP/4500 floated; **native-ESP P0.8c = UDP/500**), Quick Mode dưới SA mới → ESP CHILD SA mới, `SwapSession` (tái dùng y hệt rekey Phase 2 — **carrier giữ nguyên**, chỉ đổi keys/SPI), Delete ISAKMP SA cũ, rồi swing `_ike` sang SA mới. **Rẽ nhánh theo carrier** để CHILD SA mới khớp carrier: native-ESP dùng **identity thật** (`natt.GetLocalAddress()`) + `IkeV1Client.PreferNativeTransport` ⇒ gateway no-NAT cài SA mới **native (Transport, không espinudp)**; forced giữ `IPAddress.Any` (claim NAT, stay floated). Reply rekey tách khỏi DPD SA cũ trên cùng socket nhờ cookie SA mới (`IkeV1Client.IsForThisSa` → `_phase1RekeyWaiter`); chung guard `_rekeyInProgress` với rekey Phase 2; rekey lỗi ⇒ giữ SA cũ + retry 2 phút trong biên trước hạn cứng. Không rớt tunnel suốt rekey — [L2tpIpsecConnection.cs:761-832](L2tpIpsecConnection.cs#L761-L832). **VALIDATE LIVE ✓** (forced NAT-T + **native-ESP**, lab `l2tp-nonat`): IKE_SA `[1]→[2]` + CHILD SA mới (native giữ Transport/proto-50), địa chỉ PPP `10.31.0.10` không đổi.
- **Multi-session (P1.7, best-effort):** `OpenAdditionalSessionAsync` mở thêm session L2TP (`L2tpClient.OpenSessionAsync`) trên cùng IKE/IPsec SA + control channel rồi chạy một `PppEngine` riêng → địa chỉ/kênh thứ hai; teardown gửi CDN cho mọi session phụ trước CDN/StopCCN session chính. Đa số server từ chối ICRQ thứ hai bằng CDN (⇒ exception); session phụ **không** được tái lập khi auto-reconnect (cleanup xóa `_extraSessions`) — [L2tpIpsecConnection.cs:168-197](L2tpIpsecConnection.cs#L168-L197).
- **Link-loss + supervisor (F.6, gom về base):** mọi nguồn (DPD chết, server Delete, L2TP teardown) gọi `OnLinkLost` của base ([ReconnectingVpnConnection.cs:161](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs#L161)) → `ReconnectLoopAsync` ([@ :189](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs#L189)) với exponential backoff + jitter; reconnect thành công gọi `OnReconnected` (override — [L2tpIpsecConnection.cs:869](L2tpIpsecConnection.cs#L869)) raise `Reconnected` → cập nhật `TunnelConfig` qua `L2tpIpsecVpnSession.ApplyReconnect`.
- **Teardown:** `DisconnectAsync` (override — [L2tpIpsecConnection.cs:911](L2tpIpsecConnection.cs#L911)) gửi L2TP CDN + StopCCN và IKE Delete (ESP + ISAKMP) trên transport còn sống (best-effort, 2s), rồi gọi `DisconnectCoreAsync()` base (hủy reconnect + await supervisor + cleanup transport).
- **Logging/diagnostics (Q.2):** ctor `L2tpIpsecConnection`/`L2tpIpsecDriver` nhận `ILoggerFactory?` (mặc định [`NullLogger`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs) ⇒ no-op, **ADDITIVE không đổi hành vi**). Log qua [`VpnLogExtensions`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs): Phase 1 (MM/auth) + Quick Mode + L2TP + PPP auth-success → Handshake (auth/QM fail → HandshakeFailed); bind facade → HandshakeCompleted; L2TP HELLO + IKE DPD → Keepalive; rekey Phase 2/Phase 1 → Rekey; `SetState` (base) → StateChanged; `OnLinkLost` (base) → LinkLost; reconnect attempt/success (base) → ReconnectAttempt/Reconnected.
- **Facade ổn định:** `Facade` của base ([`SwappablePacketChannel`](../TqkLibrary.VpnClient.Abstractions/Channels/SwappablePacketChannel.cs), do `ReconnectingVpnConnection` sở hữu) cho phép tráo kênh PPP bên dưới khi reconnect mà socket trong tunnel ở tầng trên không bị đứt (nếu địa chỉ giữ nguyên).

## Trạng thái & ghi chú

- **Đã hiện thực đầy đủ và chạy live (VPN Gate):** driver L2TP/IPsec dùng **IKEv1** (`Ipsec/Ike/V1`) với vòng đời keepalive/rekey/teardown/auto-reconnect.
- **Supervisor F.6 (gom về Drivers.Core):** `L2tpIpsecConnection : ReconnectingVpnConnection<L2tpIpsecConnectionState>` + `L2tpIpsecReconnectOptions : VpnReconnectOptions` — đây là driver **cuối cùng** migrate sang base, F.6 nay xong cho mọi driver. Supervisor/reconnect/backoff/StateChanged/link-loss/teardown chung gom về base; rekey Phase1/Phase2 + multi-session + DPD/HELLO + swap SA giữ trong driver trên timer riêng (ngoài supervisor). **Không đổi hành vi** — test driver + L2tp/Ipsec.Ike + Drivers.Core còn xanh.
