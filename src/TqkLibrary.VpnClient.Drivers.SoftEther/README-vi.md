# TqkLibrary.VpnClient.Drivers.SoftEther

Driver **SoftEther SSL-VPN** (V4.c) — **driver L2 thật đầu tiên**: ráp control handshake ([`SoftEtherHandshake`](../TqkLibrary.VpnClient.SoftEther/SoftEtherHandshake.cs), V4.b) + data plane Ethernet-over-TLS ([`SoftEtherEthernetChannel`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherEthernetChannel.cs), V4.c) thành một tunnel sống sau facade, trên **một byte-stream TLS** (SoftEther = "Ethernet over HTTPS"; cùng một stream chạy control rồi data). Sau login (`welcome`), stream chuyển sang **session data**: mỗi block = `uint32(num_frames)` + mỗi frame `uint32(size)·bytes` (raw Ethernet frame hoặc keep-alive). Data channel là **`IEthernetChannel`** cắm vào L2 fabric với **2 chế độ**: (mặc định) bắc-cầu-**1-host** — [`DhcpV4Configurator`](../TqkLibrary.VpnClient.Ethernet/DhcpV4Configurator.cs) (L2.5) lấy IP từ **SecureNAT qua DHCP** → [`ArpResolver`](../TqkLibrary.VpnClient.Ethernet/ArpResolver.cs) (L2.3) + [`VirtualHost`](../TqkLibrary.VpnClient.Ethernet/VirtualHost.cs) (L2.2) xuống `IPacketChannel` ([`SwappablePacketChannel`](../TqkLibrary.VpnClient.Abstractions/Channels/SwappablePacketChannel.cs)); (opt-in `multiHost`) **multi-host broadcast domain** — data channel gắn vào [`EthernetAdapter.ConnectUplink`](../TqkLibrary.VpnClient.Ethernet/EthernetAdapter.cs) như **uplink port** + [`MultiHostSession`](../TqkLibrary.VpnClient.Ethernet/MultiHostSession.cs) (L2.8), mỗi station một MAC/IP DHCP lease riêng do SecureNAT cấp, thêm station qua `OpenSessionAsync` — IP stack chỉ thấy gói IP trần. **Opt-in `enableIpv6`**: ngoài IPv4 (DHCP+ARP), bridge chạy thêm **SLAAC/DHCPv6** ([`Ipv6AddressConfigurator`](../TqkLibrary.VpnClient.Ethernet/Ipv6AddressConfigurator.cs) L2.6) + **NDISC v6** ([`NdiscResolver`](../TqkLibrary.VpnClient.Ethernet/NdiscResolver.cs) L2.4) trên cùng segment SecureNAT — gom qua [`DualStackNeighborResolver`](../TqkLibrary.VpnClient.Ethernet/DualStackNeighborResolver.cs)/[`DualStackAddressConfigurator`](../TqkLibrary.VpnClient.Ethernet/DualStackAddressConfigurator.cs) (L2.9), đẩy `AssignedAddressV6`/`PrefixLengthV6` lên `TunnelConfig`; **best-effort** (server IPv4-only vẫn kết nối, mặc định tắt để giữ wire IPv4-only như cũ). Auth SHA-0 `secure_password` (V4.b), supervisor/auto-reconnect qua base [`ReconnectingVpnConnection`](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs) dùng chung (F.6 — cùng mẫu OpenConnect/OpenVPN/WireGuard).

## Vị trí kiến trúc

`DRIVER`-layer, hiện thực [`IVpnProtocolDriver`](../TqkLibrary.VpnClient.Abstractions/Drivers/Interfaces/IVpnProtocolDriver.cs). Lắp ráp khối protocol thuần từ [`SoftEther`](../TqkLibrary.VpnClient.SoftEther) + L2 fabric từ [`Ethernet`](../TqkLibrary.VpnClient.Ethernet) (mirror [`Drivers.OpenVpn`](../TqkLibrary.VpnClient.Drivers.OpenVpn) tap-mode + [`Drivers.WireGuard`](../TqkLibrary.VpnClient.Drivers.WireGuard)):

- **Transport**: byte-stream TLS qua seam [`ISoftEtherTransportFactory`](Transport/ISoftEtherTransportFactory.cs) — production [`SoftEtherTlsTransportFactory`](Transport/SoftEtherTlsTransportFactory.cs) dựng **shared** [`TlsByteStream`](../TqkLibrary.VpnClient.Transport.Tls/TlsByteStream.cs#L25) (F.1 — project `Transport.Tls`, TCP + `SslStream`), test inject loopback in-memory.
- **Control plane**: [`SoftEtherHandshake.RunAsync`](../TqkLibrary.VpnClient.SoftEther/SoftEtherHandshake.cs) (watermark POST → hello → login + SHA-0 → welcome) trên cùng stream.
- **Data plane**: [`SoftEtherDataBlockReader`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherDataBlockReader.cs) decode block vào → `SoftEtherEthernetChannel.Deliver`; [`SoftEtherEthernetChannel.WriteFrameAsync`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherEthernetChannel.cs) encode block ra.
- **L2→L3 bridge**: (1-host) `DhcpV4Configurator` (lease IP) → `ArpResolver` + `VirtualHost` → `IPacketChannel`; (multi-host, opt-in) `EthernetAdapter.ConnectUplink` + `MultiHostSession` → N station, mỗi station DHCP lease riêng.
- **Lifecycle**: keep-alive timer (gửi block keep-alive magic `0xFFFFFFFF`) + supervisor/auto-reconnect qua base [`ReconnectingVpnConnection`](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs) (F.6).

## Luồng kết nối (as-built)

1. **Transport TLS**: `ISoftEtherTransportFactory.ConnectAsync(host, port, afp, ct)` dựng shared `TlsByteStream` (`IByteStreamTransport`) → `transport.ConnectAsync` (TCP + TLS handshake). Resolve host nằm trong transport (`TlsByteStream` bọc `Transport.Tcp` `TcpByteStream` dùng `IHostResolver`).
2. **Control handshake** ([`SoftEtherHandshake.RunAsync`](../TqkLibrary.VpnClient.SoftEther/SoftEtherHandshake.cs#L179)): watermark POST `/vpnsvc/connect.cgi` (body `"VPNCONNECT"` hoặc blob THẬT) → đọc `hello` (random 20B challenge) → POST `login` PACK (`secure_password` SHA-0 + session params, `use_encrypt`=on) `/vpnsvc/vpn.cgi` → đọc `welcome` (DATA `session_key` + `max_connection`/`half_connection`) hoặc lỗi (`error`≠0 ⇒ `SoftEtherProtocolException`); trả thêm **leftover** byte đọc dư của handshake (re-inject trước data stream).
3. **Multi-connection (V.4)** ([`OpenAdditionalConnectionsAsync`](SoftEtherConnection.cs)): mở `min(login.MaxConnection, welcome.MaxConnection, 32)−1` socket phụ — mỗi socket TLS mới chạy [`SoftEtherHandshake.RunAdditionalConnectAsync`](../TqkLibrary.VpnClient.SoftEther/SoftEtherHandshake.cs#L212) reattach bằng session key (lỗi = best-effort, log + chạy tiếp). Gộp tất cả vào [`SoftEtherMultiConnectionMux`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherMultiConnectionMux.cs) (hướng theo [`SoftEtherConnectionDirectionPlanner`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherConnectionDirectionPlanner.cs), full/half-duplex).
4. **Data channel L2** ([`SoftEtherEthernetChannel`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherEthernetChannel.cs)): gắn lên mux — egress `mux.SendBlockAsync` (round-robin N socket), ingress `mux.StartReceiveLoops` (1 decode loop/socket) raise `mux.InboundFrame → channel.Deliver` (drop keep-alive). EOF/fault ở bất kỳ socket ⇒ link lost một lần (`OnLinkLost` ở base).
5. **DHCP lease** ([`DhcpV4Configurator`](../TqkLibrary.VpnClient.Ethernet/DhcpV4Configurator.cs)): gắn `dhcp.HandleInboundFrame` vào `channel.InboundFrame`, chạy `ConfigureAsync` (DISCOVER→OFFER→REQUEST→ACK qua L2 broadcast) → `TunnelConfig` (yiaddr + mask→prefix + router→route + DNS). SecureNAT là DHCP server. Không lease IPv4 ⇒ `VpnConnectionException`.
6. **Bridge L2↔L3**: `ArpResolver(mac, leasedIp, channel)` + `VirtualHost(mac, channel, resolver)`; nối `InboundNonIpFrame→arp` (ARP) + `InboundIpPacket→dhcp` (DHCP renewal). `config.Mtu = host.Mtu` (link−14). `Facade.SetInner(virtualHost)` → `MarkConnected()` → state Connected.
6b. **IPv6 (opt-in `enableIpv6`)**: link-local = `fe80::/64` + EUI-64(MAC) (`SlaacAddress`); dựng `NdiscResolver` + `Ipv6AddressConfigurator` trên cùng channel → `DualStackNeighborResolver(arp, ndisc)` làm resolver của `VirtualHost`; `ConfigureIpv6BestEffortAsync` gắn tạm `ndisc/ipv6Config.HandleInboundFrame` vào `channel.InboundFrame` (VirtualHost chưa bound) → RS→RA (SLAAC tạo địa chỉ global từ prefix /64 autonomous) hoặc DHCPv6 stateful (M flag), O flag kéo thêm DNS → đẩy `AssignedAddressV6`/`PrefixLengthV6`/DNS/route `::/0` lên `config`; thất bại (server IPv4-only) ⇒ bỏ qua leg v6, giữ IPv4. Sau khi bound, NDISC/DHCPv6 ride `VirtualHost.InboundIpPacket`.
7. **Keep-alive**: timer 5s gửi **block keep-alive** (`uint32(KEEP_ALIVE_MAGIC=0xFFFFFFFF)·uint32(size)·random`, [`SoftEtherDataFrameCodec.EncodeKeepAlive`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherDataFrameCodec.cs)) qua mux — block riêng (không phải Ethernet frame), reader bỏ qua; giữ session mở qua middlebox.

## Vòng đời

- **Gửi data** (`VirtualHost.WriteIpPacketAsync`): đọc dest IP → `ArpResolver.ResolveAsync` (next-hop MAC) → wrap Ethernet frame → `SoftEtherEthernetChannel.WriteFrameAsync` → encode block 1-frame → `mux.SendBlockAsync` (round-robin các socket gửi-được) → `transport.WriteAsync`.
- **Nhận data** (N decode loop trong mux): mỗi socket nhận-được decode block → `mux.InboundFrame` → `channel.Deliver(frame)` raise `InboundFrame` → `VirtualHost` strip header → `InboundIpPacket` (IP) hoặc `InboundNonIpFrame` (ARP) → `ArpResolver`.
- **Teardown** (`DisconnectAsync` ⇒ base `DisconnectCoreAsync`): hủy reconnect đang chờ, hủy receive loop + keep-alive timer (`StopAttemptLoop`), dispose `VirtualHost`/`ArpResolver`/`DhcpV4Configurator`/transport qua `CleanupAttemptResourcesAsync` (host dispose channel theo).
- **Reconnect** (base supervisor F.6): `EstablishAsync`/`ReconnectLoopAsync` backoff+jitter ở [`ReconnectingVpnConnection`](../TqkLibrary.VpnClient.Drivers.Core/ReconnectingVpnConnection.cs); DHCP lease lại nên `OnReconnected` raise `Reconnected` mang `AddressChanged` thật.
- **Logging/diagnostics (Q.2)**: ctor `SoftEtherConnection`/`SoftEtherDriver` nhận `ILoggerFactory?` (mặc định [`NullLogger`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs) ⇒ no-op, **ADDITIVE không đổi hành vi**). Log qua [`VpnLogExtensions`](../TqkLibrary.VpnClient.Abstractions/Diagnostics/Extensions/VpnLogExtensions.cs): control handshake + DHCP lease + bridge bind→Handshake; sau khi bind facade→HandshakeCompleted; keep-alive frame→Keepalive; `SetState`→StateChanged; `OnLinkLost`→LinkLost; reconnect attempt/success→ReconnectAttempt/Reconnected.

- **Đã có (V.4 payload `use_compress` + `use_encrypt`=data-trong-SSL)**: khi `login.Session.UseCompress` ⇒ dựng `SoftEtherEthernetChannel` với cờ `useCompress` (nén/giải-nén DEFLATE per-frame, [`SoftEtherPayloadCompressor`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherPayloadCompressor.cs)). **`use_encrypt` mặc định BẬT** (`SoftEtherSessionParams.UseEncrypt = true`) và driver **bắt buộc** nó: server thật chỉ giữ data plane **trong SSL** (`UseSSLDataEncryption`, Cedar `Protocol.c`) khi `use_encrypt` bật **và** fast-RC4 tắt — đúng cách driver chở data (đọc/ghi qua chính byte-stream TLS). `use_encrypt=off` ⇒ SoftEther rớt data plane xuống **raw TCP socket** (`Send/Recv secure=false`) mà transport byte-stream không biểu diễn được ⇒ session không chở data (lộ qua live: SslStream `corrupted frame`). `use_fast_rc4` (raw TCP + RC4) **không bao giờ** request ⇒ KHÔNG có lớp RC4 dưới block framing (`SoftEtherEncryptedTransport` giữ lại cho mode fast-RC4 tương lai, hiện chỉ test standalone). Test offline `SoftEtherConnectionTests.Connect_WithEncryptAndCompress_RoundTripsIp` round-trip qua server SecureNAT giả lập (loopback pipe = lớp "SSL").
- **Đã có (V.4-ipv6 — IPv6 trong tunnel, opt-in `enableIpv6`)**: ngoài IPv4 (DHCP+ARP), bridge SecureNAT chạy thêm **SLAAC/DHCPv6** (`Ipv6AddressConfigurator` L2.6) + **NDISC v6** (`NdiscResolver` L2.4) trên cùng segment, gom qua `DualStackNeighborResolver`/`DualStackAddressConfigurator` (L2.9); link-local `fe80::/64`+EUI-64(MAC) (`SlaacAddress`), `ConfigureIpv6BestEffortAsync` chạy RS→RA→SLAAC (+ DHCPv6 DNS) đẩy `AssignedAddressV6`/`PrefixLengthV6`/DNS/route lên `TunnelConfig`. **Best-effort**: server IPv4-only ⇒ leg v6 bỏ qua, IPv4 vẫn chạy; **mặc định tắt** ⇒ wire IPv4-only như cũ. Cả 1-host (`EstablishSingleHostAsync`) lẫn multi-host (`AddStationAsync`, dual-stack station). `AssignedAddressV6`/`IsIpv6Enabled` + overload `UseSoftEther(hub, enableIpv6)`. Test offline `SoftEtherConnectionTests` (1-host: SLAAC địa chỉ /64 + DHCPv6 DNS + route `::/0` + round-trip v4 **và** v6 qua NDISC thật; server IPv4-only + `enableIpv6` ⇒ vẫn Connected, `AssignedAddressV6==null`, v4 round-trip; multi-host primary station configure v4+v6) — server SecureNAT giả lập đóng vai IPv6 router (RA)+DHCPv6+NDISC NA.
- **Đã có (watermark — KHÔNG cần blob GPL)**: ctor `SoftEtherDriver(..., watermark)` → `SoftEtherConnection` nạp **watermark blob** runtime (demo `--watermark <file>`). Server thật chấp nhận POST khi body bằng hằng giao thức **`"VPNCONNECT"`** (`HTTP_VPN_TARGET_POSTDATA`, 10 byte ASCII — đường thứ hai trong `ServerDownloadSignature` của chính server) HOẶC khớp byte-exact ảnh GPL `WaterMark[]`. Lab dùng `"VPNCONNECT"` (protocol constant, KHÔNG phải ảnh có bản quyền) ⇒ **gỡ blocker watermark mà không cần/commit blob GPL** (đã xác nhận live: `"VPNCONNECT"`→200, placeholder→403). `SoftEther.Tests`=68, `Drivers.SoftEther.Tests`=23 xanh.
- **Bug interop đã sửa (phát hiện qua LIVE — offline self-pair không bắt vì cả 2 phía dùng chung codec sai):**
  - **HTTP-PACK framing**: body HTTP = PACK bytes **verbatim** (Content-Length phân định), KHÔNG có `uint32` length-prefix thừa (server `HttpClientSend`/`HttpServerSend` POST `PackToBuf(p)` trực tiếp — Mayaqua `Network.c`).
  - **Field session key trong welcome**: session key đúng là DATA **`session_key`** (20B, dùng cho `additional_connect`) + INT **`session_key_32`** — KHÔNG phải `session_name` (STR, chỉ là tên session). Trước đó đọc nhầm `session_name` ⇒ "missing session key".
  - **Data-plane SSL**: bật `use_encrypt` để data ở trong SSL (xem mục payload trên) — bỏ lớp RC4 bọc thừa.
  - **Keep-alive block**: gửi/nhận keep-alive dạng block riêng `uint32(KEEP_ALIVE_MAGIC=0xFFFFFFFF)·uint32(size)·bytes` ([`SoftEtherDataFrameCodec.EncodeKeepAlive`](../TqkLibrary.VpnClient.SoftEther/DataChannel/SoftEtherDataFrameCodec.cs)), reader bỏ qua — KHÔNG phải block 1-frame chứa chuỗi (server switch sẽ tưởng là Ethernet frame; reader cũ ném khi thấy count `0xFFFFFFFF`).
  - **Re-inject leftover handshake→data** ([`PrefixedByteStreamTransport`](../TqkLibrary.VpnClient.SoftEther/PrefixedByteStreamTransport.cs)): server có thể gộp welcome + block data đầu vào 1 TLS record ⇒ giữ byte đọc dư của handshake, đẩy lại trước data stream (bỏ ⇒ desync block reader).
- **Residual (không chặn driver)**: TCP-tới-internet qua SecureNAT chưa kiểm live (lab bridge isolated + dính residual shared-IpStack TCP send-window như V.2); multi-connection/IPv6 pool với SecureNAT thật chưa kiểm (lab single-connection IPv4). Mode `use_fast_rc4` (raw TCP + RC4) chưa hiện thực (cần phơi raw TCP dưới TLS).
