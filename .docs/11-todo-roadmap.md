## P1 — SSTP + L2TP/IPsec (kể cả IPv6) — ĐÓNG: client 100%, validate live

> **P1 đã đóng (2026-06-23).** Phần CLIENT của cả 4 mục XONG + validate live: P1.1 IPv6-in-tunnel, P1.2 outer-IPv6,
> P1.3 rekey Phase 1 (cả forced-NAT-T lẫn native-ESP), P1.7 multi-session. **Không còn việc code client.** As-built đầy đủ ở
> [`10`](10-codebase-architecture-and-flow.md) §9 + bảng "Khác biệt so với design docs" + README Sstp/L2tp/L2tpIpsec.
>
> **Residual external — không phải gap client (giữ lại để ghi nhận giới hạn server-side đã chứng minh):**
> - **L2TP/PPP-over-IPv6 external-infeasible — đã CHỨNG MINH:** không lên được vì **mọi L2TPv2 LNS có sẵn bind L2TP IPv4-only**
>   — xl2tpd 1.3.16 (`set_ip` từ chối IPv6) **và** accel-ppp 9ddf513 (L2TP control listener = `socket(PF_INET)`/`sockaddr_in`,
>   **0** `sockaddr_in6` trong `l2tp.c`, dù module SSTP/ipv6/radius của chính accel-ppp CÓ IPv6). **Carrier-độc-lập**: gói
>   L2TP/UDP sau giải mã ESP (transport mode) mang outer-addr IPv6 bất kể carrier (native-ESP/ESP-in-UDP) ⇒ **không client nào
>   mở khóa được** — chặn ở tầng LNS. (Phần SSTP-over-IPv6, IKE/ESP-over-IPv6 của client đã validate live.)
> - **P1.7 cross-server multi-session** (SoftEther/RRAS, hoặc server giới-hạn-1-phiên mong CDN-reject ⇒ `VpnServerRejectedException`):
>   xác nhận **tùy chọn** — client đã validate live (xl2tpd 2-call/1-tunnel) + offline reject-path; **không phụ thuộc code client.**

## V★ — Backlog phủ TOÀN BỘ giao thức VPN opensource / public-doc (RFC)

> Mục tiêu "**clone mọi VPN opensource + giao thức có RFC/public-doc**". Đây là backlog đầy đủ các giao thức **chưa khởi động**, gom theo họ, đánh **Tier** (1 = đòn bẩy cao/rẻ, tái dùng tối đa; 3 = niche/đắt/legacy). Mỗi mục clean-room **đọc spec, KHÔNG copy GPL/AGPL**.
