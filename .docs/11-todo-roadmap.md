## ✅ Đã xong (baseline)
- Driver **SSTP** live (TLS + PPP/MS-CHAPv2 + crypto binding) và **L2TP/IPsec** live (IKEv1 PSK + ESP + L2TP + PPP).
- **Tách project driver**: `TqkLibrary.Vpn.Drivers` cũ đã chia thành 2 project anh em **`TqkLibrary.Vpn.Drivers.L2tpIpsec`** + **`TqkLibrary.Vpn.Drivers.Sstp`** (file phẳng ở gốc project, giữ `Enums/`+`Models/`); namespace giữ nguyên. SSTP nay **chỉ ref Abstractions + Ppp** (không còn kéo Ipsec/L2tp).
- **Gom `TqkLibrary.Vpn.Transport.Udp` vào `Ipsec`**: NAT-T framing/channel (RFC 3948) nay nằm ở [`Ipsec/Nat/`](../src/TqkLibrary.Vpn.Ipsec/Nat) (namespace `TqkLibrary.Vpn.Ipsec.Nat`, enum tách vào `Nat/Enums/`); project src + test cũ đã xóa, test `NatTraversalTests.cs` gộp vào `Ipsec.Esp.Tests`; driver L2TP/IPsec bỏ ref Transport.Udp. Họ project tương lai `Transport.Tls`/`Transport.Dtls`/`Transport.RawIp` (P3) không đổi.
- **IKEv2** (`Ipsec/Ike/V2/`) đầy đủ + test, **chưa wire vào driver**.
- Socket API trong tunnel: **TCP** (`VpnTcpClient` → HttpClient) + **UDP** (`VpnUdpClient` → DNS-over-tunnel), đều live.
- **IP stack dual-stack IPv4 + IPv6** (L3): demux theo version-nibble; IPv6 codec header + extension-header walk + Fragment-header reassembly + fragmentation chiều gửi + **ICMPv6** (echo/ping + dest-unreachable + port-unreachable) + pseudo-header checksum v6; TCP/UDP **chạy y hệt** 2 họ (chỉ lớp IP bọc + checksum khác). ND/SLAAC chưa (point-to-point L3 không cần — để tầng **L2 Ethernet**).
- NAT-T (forced 500→4500), userspace IPv4/TCP/UDP, anti-replay ESP.
- **Robustness L2TP/IPsec**: keepalive (HELLO + DPD), Phase 2 rekey in-place (make-before-break), teardown sạch (CDN/StopCCN/DELETE) + `DisconnectAsync`/`IAsyncDisposable`, **auto-reconnect** (backoff, stable channel) + event `StateChanged`.
- **Robustness SSTP** (mirror L2TP/IPsec): active keepalive (Echo-Request 30s, chết sau 3 lần thiếu Echo-Response), teardown sạch (Call-Disconnect) + `IAsyncDisposable`, **auto-reconnect** (backoff, stable channel) + event `StateChanged`/`Reconnected`. **Không** rekey (TLS sống dài).
- **Typed exception** `VpnConnectionException` + 3 lớp con (auth/server-reject/network-timeout) wire ở cả 2 driver.
- 10 project `src/` + 10 project `tests/`, build xanh `netstandard2.0`+`net8.0`, **226 test offline pass** (`net8.0`) — gồm **IPv6/ICMPv6 dual-stack** (codec header + extension-header walk + Fragment-header reassembly + ICMPv6 ping/dest-unreachable + TCP/UDP over v6 + MSS=MTU−60 + closed-port RST/ICMPv6), IKEv1 capstone, TCP loopback, **TCP reliability** (retransmit/RTO + give-up + flow-control + zero-window persist + out-of-order reassembly + half-close FSM), **đàm phán MSS** (SYN quảng bá MSS=MTU−40, clamp theo MSS peer, fallback 536), **PMTUD động** (RFC 1191/8201: ICMP frag-needed/Packet-Too-Big hạ MSS dưới link MTU + re-segment retx queue), **window scaling RFC 7323** (SYN mang WS option, scale cửa sổ peer >64 KB, no-WS→unscaled), **congestion control NewReno** (slow-start IW + growth, 3 dup-ACK fast retransmit, fast-recovery inflation), **SACK RFC 2018/6675** (đàm phán SACK-Permitted, phía nhận sinh SACK blocks từ out-of-order, phía gửi retransmit chọn lọc đúng hố + pipe fill, fallback NewReno), **IPv4 fragment reassembly** (in-order/out-of-order/trùng/timeout/over-cap + UDP phân mảnh end-to-end), **IPv4 fragmentation chiều gửi** (cắt-trên-ranh-8-byte + round-trip fragment↔reassemble + UDP to qua stack), **ICMP echo/ping + destination-unreachable**, **phản hồi port đóng** (UDP→ICMP port-unreachable + TCP→RST), fuzz parser, L2TP control-channel retransmit-cap, ESP sequence-exhaustion rekey watermark, ESP suite negotiation (CBC/GCM, V1+V2); thêm ~12 live integration `[Trait("Category","Integration")]` (chạy offline bằng `--filter "Category!=Integration"`).
- **Demo tích hợp proxy** `demo/Vpn2ProxyDemo` — chi tiết as-built tách riêng ở [`12-demo-vpn2proxy.md`](12-demo-vpn2proxy.md): adapter `IProxySource` (inline) → HTTP/SOCKS proxy local qua tunnel, giữ tới khi nhấn Enter; MS-SSTP + L2TP/IPsec.

## P3 — Giao thức & tầng tương lai (theo plan "sau v1")

- [ ] **Driver IKEv2-native** (IPsec IKEv2 VPN, không qua L2TP): hạ tầng IKEv2 đã sẵn ([IkeClient.cs](../src/TqkLibrary.Vpn.Ipsec/Ike/V2/IkeClient.cs)), cần driver + CP để cấp IP trực tiếp + ESP data plane (đã có).
- [ ] **Tầng L2 Ethernet (multi-host, LAN ảo)** — đây mới là "client thật sự có LAN" (ARP/ND/broadcast giữa nhiều host); L3 hiện tại là point-to-point, không có LAN. Bám design [`00`](00-architecture-overview.md)/[`03`](03-multihost-l2-vs-l3.md)/[`09`](09-userspace-ipstack.md): `EthernetAdapter` = `EthernetSwitch` + N×`VirtualHost`{MAC, `INeighborResolver`, `IAddressConfigurator`, `IPacketChannel`}; **quy tắc vàng**: IpStack chỉ bind `IPacketChannel`, mọi ARP/MAC/DHCP nằm trong `EthernetAdapter`. **Mọi phase phủ cả IPv4 lẫn IPv6.** Chia nhỏ theo **thứ tự phụ thuộc**:
  - [ ] **L2.0 — Nền (codec Ethernet + slot interface)**: project mới `TqkLibrary.Vpn.Ethernet`; `EthernetFrame` codec (14 byte: dst/src MAC + EtherType + payload), `MacAddress` helper, EtherType `0x0800`(IPv4)/`0x86DD`(IPv6)/`0x0806`(ARP); khai báo `INeighborResolver` + `IAddressConfigurator` trong `Abstractions` (slot design `00` §5); implement [`IEthernetChannel`](../src/TqkLibrary.Vpn.Abstractions/Channels/Interfaces/IEthernetChannel.cs#L7) in-memory để test.
  - [ ] **L2.1 — `EthernetSwitch` (FDB MAC-learning)**: học src-MAC→port; unicast theo dest-MAC; flood broadcast `ff:ff:ff:ff:ff:ff` + unknown-unicast + IPv6 multicast `33:33:*`. In-memory, test được.
  - [ ] **L2.2 — `VirtualHost` + per-host `IPacketChannel`**: mỗi host = {MAC, adapter bọc/tháo Ethernet cho IpStack đã có (đã hỗ trợ dual-stack)}; MTU = link−14, clamp MSS; egress IpStack→wrap Ethernet (cần dest-MAC từ neighbor)→switch; ingress switch→strip→IpStack. 1 read-loop/VirtualHost (design `09`). **Phụ thuộc L2.1 + Ip stack dual-stack (đã xong).**
  - [ ] **L2.3 — ARP (IPv4 neighbor)** `INeighborResolver`: codec ARP request/reply (RFC 826), cache, resolve next-hop IP→MAC khi egress, trả lời ARP hỏi IP của ta. **Phụ thuộc L2.2.**
  - [ ] **L2.4 — NDISC (IPv6 neighbor) = "full ND"** `INeighborResolver` v6: codec ICMPv6 NS/NA (tái dùng [`Icmpv6`](../src/TqkLibrary.Vpn.IpStack/Icmpv6.cs#L7) đã có), solicited-node multicast, neighbor cache, DAD; parse RS/RA lấy gateway/prefix (RFC 4861). **Phụ thuộc L2.2.**
  - [ ] **L2.5 — DHCPv4 client** `IAddressConfigurator`: DISCOVER/OFFER/REQUEST/ACK qua Ethernet/UDP broadcast → lease IP/gw/DNS (RFC 2131); khớp SoftEther SecureNAT virtual-DHCP ([`07`](07-softether.md)). **Phụ thuộc L2.2 + L2.3.**
  - [ ] **L2.6 — SLAAC + DHCPv6 client** `IAddressConfigurator` v6: SLAAC từ RA (RFC 4862) + DHCPv6 stateful tùy chọn (RFC 8415). **Phụ thuộc L2.4.**

## Gợi ý thứ tự
1. ~~**Keepalive + rekey + teardown + reconnect** (P1)~~ — ✅ xong cho **cả 2 driver** (L2TP/IPsec: keepalive, Phase 2 rekey **theo time + sequence-exhaustion**, teardown, auto-reconnect + Phase 1 by-reconnect; SSTP: active keepalive, teardown Call-Disconnect, auto-reconnect) + ~~typed exception~~ + ~~timeout cấu hình được + L2TP retransmit cap~~. **Còn lại P1 robustness**: forced-NAT-T fallback.
2. ~~**IKEv1 capstone + TCP loopback + fuzz parser**~~ (P1 test) — ✅ xong regression offline. **Còn lại**: lab Docker thay server công khai.
3. **~~ICMP~~ + ~~TCP retransmit + sliding window + reassembly + half-close~~ + ~~AES-GCM ESP~~ + ~~IPv4 reassembly~~ + ~~phân mảnh outbound~~** (P2) — ✅ **xong cả nhóm**: AES-GCM ESP (negotiate IKEv1+IKEv2 qua `EspSuiteSelection`), ICMP (echo/ping + destination-unreachable), TCP reliability đầy đủ (slice 1 phía gửi: retransmit/RTO + flow-control + zero-window persist; slice 2 phía nhận: out-of-order reassembly + half-close FSM), IPv4 fragment reassembly inbound (RFC 791 §3.2) + **phân mảnh IPv4 chiều gửi** (RFC 791 §2.3, chokepoint `SendIp`) + **phản hồi port đóng** (RST/ICMP port-unreachable) + **đàm phán MSS theo link MTU** (clamp theo MSS peer) + **window scaling (RFC 7323)** + **congestion control NewReno (RFC 5681/6582)** + **~~SACK (RFC 2018/6675)~~** (✅ đàm phán SACK-Permitted + sinh/parse SACK blocks + retransmit chọn lọc đúng hố + pipe fill, fallback NewReno) + **~~IPv6/ICMPv6 dual-stack~~** (✅ L3 scope: codec + Fragment reassembly + ICMPv6 + pseudo-header v6 + demux version-nibble) + **~~PMTUD động~~** (✅ RFC 1191/8201: ICMP frag-needed/Packet-Too-Big hạ MSS dưới link MTU + re-segment retx queue đang in-flight). **P2 IP stack xong hết** — không còn mục dang dở.
4. **Tầng L2 Ethernet** (P3, LAN ảo multi-host) — chia phase L2.0→L2.9 theo thứ tự phụ thuộc (xem §P3): codec Ethernet → switch → VirtualHost → ARP(v4)/NDISC(v6) → DHCPv4/SLAAC+DHCPv6 → EthernetAdapter. Là "client thật sự có LAN"; ND/SLAAC sống ở đây (không phải L3).
5. **Driver IKEv2-native** (P3) — tận dụng hạ tầng IKEv2 đã build sẵn, chi phí thấp nhất trong nhóm driver P3.
6. Các driver/tầng còn lại theo nhu cầu.
