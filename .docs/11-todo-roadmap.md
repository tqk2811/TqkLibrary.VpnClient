## L2 — Tầng Ethernet còn lại (LAN ảo multi-host)

> L2.0 codec + L2.1 `EthernetSwitch` + L2.2 `VirtualHost` + L2.3 ARP + L2.4 NDISC + L2.5 DHCPv4 + L2.6 SLAAC/DHCPv6 + L2.7 `EthernetAdapter` ráp + L2.8 năng lực/multi-host-session + L2.9 composite dual-stack (`DualStackNeighborResolver`/`DualStackAddressConfigurator`) + test multi-host dual-stack offline **đã xong** (as-built: [`10`](10-codebase-architecture-and-flow.md) + [README Ethernet](../src/TqkLibrary.VpnClient.Ethernet/README-vi.md)).
> Quy tắc vàng giữ nguyên: IpStack chỉ bind `IPacketChannel`, mọi ARP/ND/DHCP nằm trong tầng L2.
> **Driver L2 thật** tiêu thụ fabric này: **OpenVPN tap-mode (V2.h)** và **SoftEther (V4.c)** — cả hai có **hai chế độ**: (1) bắc-cầu-1-host mặc định (`OpenVpnTapChannel`/`SoftEtherEthernetChannel`→`VirtualHost`+`ArpResolver`→`IPacketChannel`), và (2) **multi-host data-plane** opt-in: data channel gắn vào `EthernetSwitch`/`EthernetAdapter` như **uplink port** (`ConnectUplink`), `MultiHostSession` (L2.8) phơi N station = N `IVpnSession` (mỗi station DHCP lease riêng do server cấp), thêm station qua `IVpnConnection.OpenSessionAsync`. OpenVPN tap còn có **pure-DHCP** (server không push ifconfig → chạy `DhcpV4Configurator` L2.5). As-built ở [`10`](10-codebase-architecture-and-flow.md) §5/§8.
